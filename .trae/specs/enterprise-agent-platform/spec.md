# 《有机系统》企业级智能体平台 需求规格说明书

> Change-ID: `enterprise-agent-platform`
> 版本：v1.0
> 日期：2026-07-15
> 类型：全新平台建设

---

## 一、Why（项目背景与目标）

作为一家投资公司，企业内部拥有大量保密材料与文档（投资标的、尽调报告、内部研判、客户资料等）。市面上 SaaS 化的智能体产品无法满足"数据不出企业"的安全合规要求；同时，员工在日常工作中面临：信息检索分散、跨系统操作繁琐、重复性劳动多、专家经验难复用等痛点。

本平台命名为 **《有机系统》（AgenticOS）**，定位为企业内部安全、自主、可演化的"超级智能体平台"。核心理念是像有机体一样：能感知（理解用户）、能记忆（长期陪伴）、能生长（技能与专家生态）、能协作（MCP 接入企业服务）。对标产品：腾讯 WorkBuddy、OpenClaw、Hermes Agent。

**核心目标：**
1. 数据主权完全归属企业，支持全离线/私有化部署；
2. 长期陪伴单一员工，沉淀其工作习惯与偏好；
3. 通过"技能 + 专家 Agent + MCP 服务"三层能力体系，持续扩展员工生产力边界；
4. 员工自主选择连接的企业系统，实现"按需接入、可控可审计"。

---

## 二、What Changes（变更概览）

本变更将新建一个完整的企业级智能体平台，包含以下能力域：

### 平台级新建能力
- **用户与组织管理**：企业组织架构同步、SSO 单点登录、角色与权限。
- **长期记忆与用户画像**：构建员工专属长期记忆库与偏好画像，实现"懂你"的陪伴式交互。
- **智能体编排引擎**：负责任务规划、工具调用、多轮反思、记忆检索的统一调度核心。
- **技能系统**：用户可自定义技能（提示词模板 + 工具 + 工作流），支持版本管理与私有/共享。
- **技能公共市场**：企业级 + 公共两级技能市场，用户可一键安装所需技能。
- **专家 Agent 市场**：提供预置的领域专家 Agent（投资分析师、法务顾问、HR 助理、会议纪要专家等）。
- **MCP 服务集成层**：将企业内部 OA/CRM/文档系统/数据中台等以 MCP（Model Context Protocol）标准接口暴露，用户主动勾选连接。
- **知识库与 RAG**：企业文档向量化、检索增强、引用可溯源。
- **安全合规**：数据加密、操作审计、脱敏、权限隔离、模型网关。
- **前端入口**：Web 端 + Electron 桌面客户端（对接用户既有技术栈）。

### 标注
- 全部为 **ADDED**（全新建设），无遗留系统改造。
- 涉及 **BREAKING**：无（绿地项目）。

## 四、ADDED Requirements（新增需求详述）

> 约定：每个需求以"系统 SHALL…"形式描述，并给出关键场景（Scenario）。

---

### 需求 1：用户与组织管理

#### REQ-1.1 身份认证与单点登录
系统 SHALL 支持企业 SSO（OIDC / SAML 2.0 / LDAP）接入，员工使用企业账号免密登录；同时支持本地账号兜底。

**Scenario: SSO 登录**
- WHEN 员工首次通过企业 SSO 登录
- THEN 系统自动创建用户、绑定组织架构、分配默认角色 `employee`，并生成专属记忆空间

**Scenario: 多端会话**
- WHEN 同一员工在 Web 与 Electron 端同时登录
- THEN 两端会话隔离但共享同一用户画像与长期记忆，操作可双向同步

#### REQ-1.2 组织架构与角色
系统 SHALL 同步企业组织架构，支持角色：`employee`（普通员工）、`skill-author`（技能作者）、`admin`（平台管理员）、`auditor`（审计员）。

**Scenario: 部门隔离**
- WHEN 投资一部员工发起对话
- THEN 默认仅检索本部门可见的知识与技能；跨部门资源需显式申请

#### REQ-1.3 权限模型
系统 SHALL 实现 RBAC + ABAC 混合权限：基于角色控制功能权限，基于属性（部门/密级/数据标签）控制数据权限。

---

### 需求 2：长期记忆与用户画像（理解并陪伴用户）

> 这是平台的"灵魂"能力，对标"长期陪伴"。

#### REQ-2.1 短期对话记忆
系统 SHALL 在单次会话内维护上下文记忆，支持滑动窗口 + 摘要压缩，避免超出上下文长度。

#### REQ-2.2 长期记忆库
系统 SHALL 为每位员工建立独立的长期记忆库，包含三类记忆：
- **事实记忆**（Fact）：员工岗位、汇报关系、常用项目、关注赛道等；
- **偏好记忆**（Preference）：沟通风格（简洁/详尽）、输出格式（表格/列表）、关注维度（财务/技术/团队）；
- **事件记忆**（Episode）：历史任务、协作记录、反馈。

**Scenario: 偏好学习**
- WHEN 员工连续 3 次要求"用表格输出"
- THEN 系统将其沉淀为偏好，后续默认表格输出，并可在画像页查看/编辑

**Scenario: 跨会话记忆召回**
- WHEN 员工说"接着上次那个项目"
- THEN 系统从长期记忆检索最近相关项目上下文，自动注入提示，无需重复说明

#### REQ-2.3 记忆的写入、更新与遗忘
- 写入：采用"LLM 提取 → 冲突检测 → 合并/覆盖"流程，避免冗余；
- 更新：支持时间衰减 + 显式纠正（用户说"不是这样"则更新）；
- 遗忘：支持用户主动删除单条记忆或"清空某主题"；
- 审计：所有记忆读写留痕，可审计。

**Scenario: 纠正错误记忆**
- WHEN 系统基于旧记忆给出错误判断，用户纠正"我已调岗到消费组"
- THEN 系统更新事实记忆，并标注变更原因与时间

#### REQ-2.4 用户画像可视化
系统 SHALL 提供画像主页，员工可查看：已沉淀的事实/偏好/事件、记忆来源、可信度，并支持手动编辑与导出。

---

### 需求 3：智能体编排引擎（Agent Orchestration）

#### REQ-3.1 任务规划与执行
系统 SHALL 提供统一编排引擎，支持：
- 单轮问答；
- 多步任务规划（Plan → Act → Observe → Reflect）；
- 工具调用（ReAct / Function Calling）；
- 并行子任务与子 Agent 委派。

**Scenario: 多步尽调任务**
- WHEN 员工说"帮我分析一下 A 公司，重点看财务和团队"
- THEN 引擎拆分为：检索公司资料 → 调用财务 MCP 服务 → 调用工商 MCP 服务 → 生成结构化报告

#### REQ-3.2 多模型路由
系统 SHALL 通过模型网关支持多模型路由：默认模型 + 长文模型 + 代码模型 + 视觉模型，按任务类型与成本自动选择；支持企业自部署开源模型（如 Qwen / DeepSeek / GLM）与商用 API（合规审批后）。

#### REQ-3.3 流式输出与中断
系统 SHALL 支持流式输出，并允许用户随时中断、追问、回滚到某一步重试。

#### REQ-3.4 可观测性
系统 SHALL 记录每次 Agent 执行的完整 trace（输入/工具调用/中间思考/输出/耗时/token 成本），供用户与审计员查看。

---

### 需求 4：技能系统（用户自定义技能）

#### REQ-4.1 技能定义模型
一个技能 SHALL 由以下要素组成：
- 元数据：名称、描述、图标、作者、版本、分类、标签；
- 提示词模板（含变量占位符）；
- 输入参数 Schema（JSON Schema）；
- 依赖工具（可绑定 MCP 服务或内置工具）；
- 执行流程（可选 DAG 工作流，支持条件分支）；
- 输出格式约定；
- 权限范围（仅本人 / 本部门 / 全企业 / 公共）。

#### REQ-4.2 技能创建与编辑
系统 SHALL 提供低代码技能编辑器：可视化拖拽 + 模板编写，支持在线调试、版本回滚、A/B 测试。

**Scenario: 自定义"尽调摘要"技能**
- WHEN 员工创建一个技能："输入 = 公司名 + 尽调报告 → 输出 = 三段式摘要 + 风险点"
- THEN 员工可在编辑器中绑定"文档检索"工具、编写提示词、用样本数据试跑，确认后保存为私有技能

#### REQ-4.3 技能版本与共享
- 支持语义化版本号（SemVer）；
- 支持技能共享链接、部门共享、私有；
- 技能更新可触发已订阅用户的"更新提示"。

#### REQ-4.4 技能运行时沙箱
系统 SHALL 在隔离沙箱中执行技能中的代码类步骤（如 Python 脚本），限制文件/网络/内存访问，保障平台安全。

---

### 需求 5：技能公共市场

#### REQ-5.1 市场结构
系统 SHALL 提供两级市场：
- **企业内市场**：本企业员工发布、仅本企业可见；
- **公共市场**：由平台运营方维护的预置技能库（可联网同步，也可离线镜像导入）。

#### REQ-5.2 市场浏览与检索
- 分类导航（通用办公 / 投资研究 / 法务 / HR / 数据分析 / 会议协作…）；
- 关键词搜索 + 语义搜索；
- 评分、使用量、作者信誉排序；
- 详情页：描述、示例输入输出、依赖权限、更新日志、用户评论。

**Scenario: 一键安装技能**
- WHEN 员工在市场选中"周报生成器"技能并点击安装
- THEN 技能安装到个人技能库，依赖的 MCP 服务若未连接则提示授权连接；安装行为记入审计

#### REQ-5.3 技能审核与下架
- 企业内发布技能需经 `admin` 审核或自动安全扫描（提示词注入检测、恶意代码检测）；
- 支持紧急下架与已安装版本冻结。

---

### 需求 6：专家 Agent 市场

#### REQ-6.1 专家 Agent 模型
一个专家 Agent SHALL 包含：
- 角色设定（Persona：身份、专长、语气）；
- 系统提示词；
- 绑定技能集合；
- 绑定 MCP 服务（如法务 Agent 绑定合同系统）；
- 绑定知识库（如法规库）；
- 记忆策略（是否独立记忆、是否共享员工画像）。

#### REQ-6.2 预置专家 Agent
平台 SHALL 首期提供：
1. **投资分析专家**：财务建模、估值、行业对比；
2. **法务顾问**：合同审查、合规风险、条款解释；
3. **HR 助理**：政策问答、流程指引、简历筛选辅助；
4. **会议纪要专家**：转写、摘要、待办提取；
5. **文档摘要专家**：长文档压缩、要点提取、对比分析；
6. **数据查询专家**：自然语言转 SQL/数据 API。

#### REQ-6.3 专家 Agent 的订阅与个性化
- 员工可订阅多个专家，每个专家保留与该员工的独立长期记忆；
- 员工可"微调"已订阅专家（调整语气、追加私有知识），形成个性化变体。

**Scenario: 法务专家长期记忆**
- WHEN 员工多次与法务专家协作审查合同
- THEN 法务专家沉淀"该员工关注违约金条款、偏好逐条批注"的偏好，后续自动采用该风格

---

### 需求 7：MCP 服务集成层

> 核心：把企业内部系统统一封装为 MCP（Model Context Protocol）标准接口，用户主动勾选连接。

#### REQ-7.1 MCP Adapter 框架
系统 SHALL 提供 MCP Adapter 开发框架，将企业系统（OA / CRM / 文档中心 / 数据中台 / 邮件 / 日历 / 财务系统）封装为符合 MCP 规范的服务端，暴露：
- `tools`（可被 Agent 调用的函数）；
- `resources`（可读取的资源，如文档）；
- `prompts`（预置提示模板）。

#### REQ-7.2 服务目录与连接管理
系统 SHALL 维护"MCP 服务目录"，列出所有可连接的企业服务及其能力清单、所需权限、数据范围。员工可：
- 浏览目录；
- 主动勾选要连接的服务；
- 授权范围（只读 / 读写 / 限定数据标签）；
- 随时撤销连接。

**Scenario: 用户连接 CRM**
- WHEN 员工在"我的连接"中勾选"CRM 系统（只读：本部门客户）"
- THEN 平台为该员工建立到 CRM MCP 服务的授权通道；Agent 在后续任务中可调用 CRM 的 `search_customer`、`get_deal` 等工具，但仅限本部门数据

#### REQ-7.3 连接的运行时管控
- 所有 MCP 调用需经平台网关统一鉴权、脱敏、审计；
- 支持调用频率限制、配额管理；
- 敏感操作（写操作、删除）需员工二次确认或走审批流。

#### REQ-7.4 离线/内网部署适配
系统 SHALL 支持在纯内网部署 MCP 服务（不依赖公网），并通过服务健康检查与降级策略保障可用性。

---

### 需求 8：知识库与 RAG

#### REQ-8.1 文档接入
系统 SHALL 支持多源文档接入：本地上传、企业文档系统同步（通过 MCP）、邮件附件、网页抓取（白名单）。

#### REQ-8.2 文档处理流水线
- 解析（PDF / Word / Excel / PPT / Markdown / 图片 OCR）；
- 分块（按语义/结构，保留层级与页码）；
- 向量化（支持多 embedding 模型，可热切换）；
- 入库（向量库 + 关键词倒排，支持混合检索）；
- 元数据标注（部门、密级、来源、时间）。

#### REQ-8.3 检索与引用
- 支持语义检索 + 关键词检索 + 元数据过滤；
- 结果带置信度与来源引用（文件名 + 页码），可点击跳转原文；
- 支持"检索失败兜底"与"未命中提示"。

**Scenario: 尽调报告问答**
- WHEN 员工提问"A 公司去年的营收是多少"
- THEN 系统检索"投资一部尽调报告"分块，返回数值并标注来源 [尽调报告_A公司.pdf P.12]

---

### 需求 9：安全与合规

#### REQ-9.1 数据加密
- 传输：TLS 1.3；
- 存储：数据库 TDE、对象存储 SSE、向量库加密；
- 密钥：企业自管 KMS（支持 HSM）。

#### REQ-9.2 数据隔离
- 租户隔离（单租户私有化部署默认隔离）；
- 部门级数据隔离（向量库分片 + 元数据过滤）；
- 用户记忆库隔离（按用户 ID 强隔离）。

#### REQ-9.3 审计与可追溯
系统 SHALL 记录全链路审计日志：登录、记忆读写、技能安装、MCP 连接、Agent 执行、数据访问；支持按用户/时间/操作类型检索，可导出与告警。

#### REQ-9.4 脱敏与合规
- 输入/输出可配置脱敏规则（手机号、身份证、金额阈值）；
- 涉密文档访问需走审批；
- 支持数据驻留策略（数据不出指定机房）。

#### REQ-9.5 提示词安全
- 提示词注入检测；
- 越权调用检测（如技能尝试访问未授权 MCP）；
- 输出内容安全过滤（可选接入企业内容安全策略）。

---

### 需求 10：前端与多端入口

#### REQ-10.1 Web 端
- 对话工作台（主入口）：支持多会话、侧边栏（记忆 / 技能 / 专家 / 连接）；
- 技能市场与专家市场页面；
- 我的管理台：我的画像、我的技能、我的连接、我的订阅；
- 知识库管理；
- 管理后台：用户、组织、审计、模型、市场审核。

#### REQ-10.2 Electron 桌面客户端
- 基于用户既有 Electron 技术栈构建；
- 支持全局快捷键唤起、系统托盘、离线缓存；
- 与 Web 端账号与状态同步。

#### REQ-10.3 可访问性与性能
- 首屏加载 < 2s；对话首 token < 1.5s（P95）；
- 支持深色模式、快捷键、多语言（中/英）。

```
├─────────────────────────────────────────────────────────────┤

└─────────────────────────────────────────────────────────────┘
```

### 5.2 推荐技术栈

| 层级 | 推荐技术 | 说明 |
|------|----------|------|
| 前端 Web | React + TypeScript + Vite + Tailwind + shadcn/ui | 生态成熟、类型安全 |
| 桌面端 | Electron + 复用 Web 资源 | 对齐用户既有技术栈 |
| 后端主语言 | Python（FastAPI）或 Go | Python 利好 LLM 生态；Go 利好并发与 MCP 网关 |
| Agent 框架 | LangGraph / 自研轻量编排 | 推荐 LangGraph 做状态机式编排 |
| 模型网关 | LiteLLM / 自研 | 多模型路由、成本管控、限流 |
| LLM | 企业自部署 Qwen / DeepSeek / GLM；可选商用 API | 投资场景优先中文能力强者 |
| 向量库 | Milvus / Qdrant | Milvus 规模化能力强；Qdrant 部署轻 |
| 关系数据库 | PostgreSQL | 支持 pgvector 兜底 |
| 文档存储 | MinIO（S3 兼容） | 私有化对象存储 |
| 记忆库 | PostgreSQL + 向量库 + Redis（短期） | 三层记忆组合 |
| 消息队列 | Redis Streams / NATS | 异步任务、事件 |
| MCP | 官方 MCP SDK（Python/TS） | 标准化企业服务暴露 |
| 部署 | Docker + Kubernetes / Docker Compose（小规模） | 支持 K8s 弹性扩缩 |
| 监控 | Prometheus + Grafana + Loki | 指标 + 日志 |
| 链路追踪 | OpenTelemetry | Agent trace 可观测 |
| 认证 | Keycloak（SSO/OIDC/SAML/LDAP） | 企业级 IAM |

### 5.3 关键设计要点

1. **记忆分层架构**：短期（Redis）→ 工作记忆（会话级）→ 长期事实/偏好/事件（PG + 向量）。记忆写入走"提取-去重-合并"流水线。
2. **MCP 网关统一收口**：所有企业服务调用必经 MCP 网关，便于鉴权、脱敏、审计、限流；MCP Adapter 可独立部署在内网各系统侧。
3. **技能与专家解耦**：专家 = Persona + 技能集合 + MCP + 知识库 + 记忆策略；技能可被多个专家复用。
4. **模型可插拔**：通过模型网关抽象，企业可随时切换底层模型，不耦合业务。
5. **安全优先**：默认全私有化、最小权限、全链路审计、密钥自管。
6. **可演化**：技能市场与专家市场支持热更新，平台能力可持续生长。

### 5.4 部署形态

- **私有化单机版**（小规模 / 试用）：Docker Compose 一键部署；
- **私有化集群版**（生产）：K8s + 高可用 PG/Milvus/MinIO；
- **全离线**：模型与公共市场镜像离线导入，不依赖公网。

## 八、风险与对策

| 风险 | 对策 |
|------|------|
| 企业系统接口不标准、改造难 | 通过 MCP Adapter 适配层屏蔽差异，先接只读 |
| 模型能力不足影响体验 | 多模型路由 + 路由策略可调；保留商用 API 逃生通道 |
| 数据越权泄露 | 默认最小权限 + ABAC + 审计 + 脱敏 |
| 技能市场引入恶意内容 | 发布审核 + 安全扫描 + 沙箱执行 + 紧急下架 |
| 员工记忆隐私争议 | 画像可见可编辑可删除；明确企业数据归属策略 |
