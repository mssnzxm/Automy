    • MCP 协议与接入

    Model Context Protocol(MCP)是 Anthropic 主导的开放标准,定义 LLM 应用与外部系统的交互方式。 Automy 全面采用 MCP 作为 Agent 接入企业系统(CRM、ERP、Wind、法规库、邮件等)的统一协议, 避免为每个系统定制适配代码,实现「一次接入、全网可用」。本章节介绍 MCP 协议、Automy 接入架构、 Adapter 框架、服务目录、调用流程、限流配额与安全治理。

    🔌
    为什么选 MCP
    标准协议替代定制集成 · 工具/资源/提示三态模型 · 跨厂商中立 · 生态可复用 · 已成行业事实标准

    二、Automy MCP 接入架构

    三、MCP Gateway 职责详解

    职责说明关键设计
    服务发现从 Service Registry 获取可用 MCP Server 列表与 schemaetcd 存储 + 心跳健康检查
    鉴权校验 Agent 调用凭证与用户授权JWT(Z-token)+ OAuth2 用户令牌透传
    限流配额防止单个 Agent/用户耗尽后端资源令牌桶 + 按用户/服务/Tool 三维配额
    审计日志记录每次调用,满足金融合规异步写入 + 签名链防篡改
    脱敏过滤对返回结果中的敏感字段脱敏基于规则的字段级脱敏
    熔断降级后端异常时快速失败,保护系统滑动窗口错误率 > 阈值触发熔断
    缓存幂等 Resource 读取缓存Redis 缓存 + TTL

    五、Adapter 框架(快速接入新系统)

    Automy 提供 MCP Adapter SDK,开发者只需实现少量接口即可把企业系统接入 MCP 协议。 SDK 已封装协议层、连接管理、schema 生成、错误处理。

    Python - CRM MCP Server(完整示例) 
    from mcp import MCPServer, tool, resource
from automy.mcp.adapter import AutomyAdapter

# 1. 创建 Server,继承 AutomyAdapter 自动获得鉴权/审计/限流
server = MCPServer("crm-system", adapter=AutomyAdapter())

# 2. 暴露 Resource:客户详情(只读)
@server.resource("customer://{customer_id}")
async def get_customer(customer_id: str):
    """获取客户详情"""
    customer = await crm.get_customer(customer_id)
    return {
        "id": customer.id,
        "name": customer.name,
        "size": customer.size,
        "contracts": customer.contracts
    }

# 3. 暴露 Tool:查询商机(带副作用,需审计)
@server.tool(
    name="crm.opportunity.list",
    description="列出商机,支持金额/日期过滤",
    audit=True,
    rate_limit={"qps": 10, "daily": 1000}
)
async def list_opportunities(
    min_amount: float = 0,
    date_from: str = None,
    date_to: str = None
):
    """列出商机

    Args:
        min_amount: 最小金额(万元)
        date_from: 开始日期 YYYY-MM-DD
        date_to: 结束日期 YYYY-MM-DD
    """
    opportunities = await crm.query_opportunities(
        min_amount=min_amount * 10000,
        date_from=date_from,
        date_to=date_to
    )
    return [to_dict(o) for o in opportunities]

if __name__ == "__main__":
    server.run(transport="sse", host="0.0.0.0", port=8080)

    七、限流配额设计

    九、安全治理

    🛡 调用安全

    • Token 透传:用户 OAuth token 透传到 MCP Server,以用户身份执行
    • 权限校验:每次 tool_call 校验用户是否有该工具权限
    • 敏感操作确认:写操作(mail.send、doc.share)需用户二次确认
    • 结果脱敏:手机/身份证/银行卡字段自动脱敏

    📋 审计合规

    • 全量审计:每次 tool_call 记录用户/时间/参数/结果
    • 独立存储:审计日志写入独立 PG,签名链防篡改
    • 5 年留存:满足金融监管审计要求
    • 合规导出:支持导出标准格式供监管检查

    十一、业务场景示例

    💼 场景:销售分析师查询客户商机
    1. 销售分析师提问:「本月金额超过 500 万的商机有哪些?客户行业分布?」
    2. Agent(LLM)决策:调用 crm.opportunity.list(min_amount=500, date_from=本月)
    3. MCP Gateway 鉴权 → 限流检查 → 调用 CRM MCP Server
    4. CRM 返回 8 个商机,含客户 ID、金额、阶段
    5. Gateway 脱敏:隐藏联系人手机号中间 4 位
    6. Agent 再次调用 crm.customer.get 批量获取客户行业
    7. LLM 综合生成回答 + 表格,引用 CRM 数据来源
    8. 全程审计日志记录,可在管理后台查询调用链
