    • MCP 协议与接入

    Model Context Protocol(MCP)是 Anthropic 主导的开放标准,定义 LLM 应用与外部系统的交互方式。 Automy 全面采用 MCP 作为 Agent 接入企业系统(CRM、ERP、Wind、法规库、邮件等)的统一协议, 避免为每个系统定制适配代码,实现「一次接入、全网可用」。本章节介绍 MCP 协议、Automy 接入架构、 Adapter 框架、服务目录、调用流程、限流配额与安全治理。

    三、MCP Gateway 职责详解

    职责说明关键设计
    服务发现从 Service Registry 获取可用 MCP Server 列表与 schemaetcd 存储 + 心跳健康检查
    鉴权校验 Agent 调用凭证与用户授权JWT(Z-token)+ OAuth2 用户令牌透传
    限流配额防止单个 Agent/用户耗尽后端资源令牌桶 + 按用户/服务/Tool 三维配额
    审计日志记录每次调用,满足金融合规异步写入 + 签名链防篡改
    脱敏过滤对返回结果中的敏感字段脱敏基于规则的字段级脱敏
    熔断降级后端异常时快速失败,保护系统滑动窗口错误率 > 阈值触发熔断
    缓存幂等 Resource 读取缓存Redis 缓存 + TTL

    五、Adapter 框架(快速接入新系统)

    Automy 基于 AgentScope Java 2.0 提供 MCP Adapter 框架:通过 @Tool 注解 + Toolkit 自动生成 JSON Schema 注入提示词, 并由 AgentScope 原生 MCP 客户端连接任意 MCP 兼容服务器(文件系统/数据库/浏览器/代码解释器等)。 开发者只需标注少量注解即可把企业系统接入 MCP 协议,框架已封装协议层、连接管理、schema 生成、错误处理。

    Java - @Tool 注解 + Toolkit + MCP 客户端(完整示例) 
    // 1. 用 @Tool 注解标记方法,框架自动生成 JSON Schema 注入提示词
public class CrmTools {
    @Tool(description = "搜索客户")
    public List<Customer> searchCustomer(
        @Param("name") String name
    ) {
        return crmClient.searchByName(name);
    }

    @Tool(description = "列出商机,支持金额/日期过滤")
    public List<Opportunity> listOpportunities(
        @Param("minAmount") double minAmount,
        @Param("dateFrom") String dateFrom,
        @Param("dateTo") String dateTo
    ) {
        return crmClient.queryOpportunities(minAmount, dateFrom, dateTo);
    }
}

// 2. 封装为 Toolkit,框架自动生成 JSON Schema
var toolkit = Toolkit.of(new CrmTools());

// 3. AgentScope 原生 MCP 客户端连接远端 MCP 服务器
var mcpClient = McpClient.builder()
    .endpoint("sse://crm-mcp.automy.internal:8080")
    .auth(userToken)
    .build();
var mcpToolkit = mcpClient.asToolkit();  // 远端 MCP 工具自动转 Toolkit

// 4. 注册到 ReActAgent,LLM 自主决策调用本地/远端工具
var agent = ReActAgent.builder()
    .model(new QwenConfig("qwen-plus"))
    .tools(List.of(toolkit, mcpToolkit))
    .build();

    六、A2A 协议与 MCP 双层协作

    AgentScope Java 2.0 原生支持 A2A(Agent-to-Agent)协议,允许 Agent 注册到 Nacos 服务注册中心, 其他 Agent 可像调用微服务一样发现并委派任务,实现跨进程、跨机器的 Agent 协同。在 Automy 的 MCP 集成层中, A2A 与 MCP 形成双层协作模型:

    • 工具调用层(MCP):Agent 通过 MCP 协议调用外部系统工具(CRM/Wind/Mail 等),面向「原子能力」
    • Agent 委派层(A2A):Agent 通过 A2A 协议委派任务给其他 Agent(如投研 Subagent、法务 Subagent),面向「领域专家」

    此外,Higress AI 网关可作为 MCP 代理网关统一收口:对外暴露统一 MCP 端点,对内做鉴权、限流、审计、协议转换, 使企业内多个 MCP Server 经 Higress 收口后对外呈现为单一入口,与既有 Spring Cloud / Nacos 体系无缝融合。

    十、安全治理

    🛡 调用安全

    • Token 透传:用户 OAuth token 透传到 MCP Server,以用户身份执行
    • 权限校验:每次 tool_call 校验用户是否有该工具权限
    • 敏感操作确认:写操作(mail.send、doc.share)需用户二次确认
    • 结果脱敏:手机/身份证/银行卡字段自动脱敏

    📋 审计合规

    • 全量审计:每次 tool_call 记录用户/时间/参数/结果
    • 独立存储:审计日志写入独立 PG,签名链防篡改
    • 5 年留存:满足金融监管审计要求
    • 合规导出:支持导出标准格式供监管检查

    十二、业务场景示例

    💼 场景:销售分析师查询客户商机
    1. 销售分析师提问:「本月金额超过 500 万的商机有哪些?客户行业分布?」
    2. Agent(LLM)决策:调用 crm.opportunity.list(min_amount=500, date_from=本月)
    3. MCP Gateway 鉴权 → 限流检查 → 调用 CRM MCP Server
    4. CRM 返回 8 个商机,含客户 ID、金额、阶段
    5. Gateway 脱敏:隐藏联系人手机号中间 4 位
    6. Agent 再次调用 crm.customer.get 批量获取客户行业
    7. LLM 综合生成回答 + 表格,引用 CRM 数据来源
    8. 全程审计日志记录,可在管理后台查询调用链
