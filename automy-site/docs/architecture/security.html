    • 安全与合规

    安全是 Automy 的第一原则。面向投资公司的高合规要求,本章节介绍系统的六层纵深防御架构、 身份与权限(RBAC + ABAC)、密钥管理(KMS/HSM)、数据脱敏与分级、审计日志防篡改、 Agent 行为安全与金融行业合规支持。

    🛡
    安全设计六大原则
    数据主权归企业所有 · 最小权限原则 · 全链路审计 · 纵深防御 · 默认安全 · 私有化优先

    一、六层纵深防御架构

    Automy 采用六层纵深防御,从外到内逐层加固,任一层被突破仍有下层防护。

    三、权限模型:RBAC + ABAC

    Automy 权限模型融合 RBAC(基于角色) 与 ABAC(基于属性), 角色定义粗粒度权限,属性(部门、项目、密级)做细粒度数据控制,满足投资公司复杂的权限场景。

    维度模型示例粒度
    功能权限RBAC角色「投研分析师」含 read:knowledge操作级
    部门数据ABAC仅能访问本部门 + 公开部门文档行级
    项目数据ABAC仅能访问本人参与的项目行级
    密级控制ABAC用户 clearance ≥ 文档 access_level行级
    字段脱敏ABACL2 用户看到手机号脱敏,L4 看原文列级
    临时授权ABAC项目临时成员,授权窗口 7 天行级 + 时间

    3.1 权限策略示例(OPA Rego)

    # OPA 策略:用户能否读取某文档
package automy.doc

default allow = false

allow {
    # 1. RBAC:用户角色含 read:knowledge
    some role
    role := input.user.roles[_]
    "read:knowledge" in role.permissions

    # 2. ABAC:部门匹配(本部门或公开)
    doc := input.resource
    doc.dept_id == input.user.dept_id
    or doc.access_level == "L1"

    # 3. ABAC:密级校验
    clearance_level(doc.access_level) <= clearance_level(input.user.clearance)

    # 4. ABAC:项目成员校验
    doc.project_id in input.user.projects
    or not doc.project_id
}

clearance_level("L1") = 1
clearance_level("L2") = 2
clearance_level("L3") = 3
clearance_level("L4") = 4

    四、密钥管理(KMS / HSM)

    5.2 数据分级

    级别说明典型数据处理要求
    L4 绝密核心交易数据、客户清单未公开交易流水、客户资产禁止 Agent 访问,仅人工
    L3 机密研究报告、尽调材料未公开研报、IPO 尽调需授权访问,全程审计
    L2 内部内部文档、流程内部流程、会议纪要登录可见
    L1 公开公开资料公开年报、行业新闻无限制

    七、Agent 行为安全

    Agent 拥有自主决策能力,行为安全是 LLM 应用独有的风险维度。Automy 通过四道防线约束 Agent。

    Agent 行为安全四道防线 Mermaid

    敏感操作需用户确认"] F2 -->|确认| EXEC["执行"] EXEC --> F3["3. 输出过滤
    敏感词/越权内容检测"] F3 -->|通过| F4["4. 审计记录

    风险场景防护措施
    Prompt 注入用户输入恶意指令劫持 Agent输入过滤 + 系统提示隔离 + 输出校验
    越权调用Agent 调用无权限工具每次 tool_call RBAC 校验
    数据泄露Agent 把 L3 数据输出给 L2 用户输出按用户密级过滤
    恶意代码技能代码含后门沙箱隔离 + 静态扫描
    幻觉越权Agent 编造未授权数据RAG 强制引用 + 数据源校验
    敏感操作Agent 自动发送邮件/转账写操作强制用户二次确认

    八、合规支持

    🏦 金融行业

    符合证券、基金、私募股权监管要求,支持合规审计。满足《证券公司信息隔离墙指引》。

    🔒 数据保护

    符合《数据安全法》《个人信息保护法》要求,数据分级、最小化、可删除。

    📋 等保合规

    支持等保 2.0 三级要求,可对接等保测评,提供安全合规报告。

    📜 反洗钱

    支持交易数据留痕、异常行为监测,满足反洗钱合规要求。

    🔇 信息隔离

    支持业务隔离墙,投研/自营/经纪数据物理隔离,防止内幕信息泄露。

    📋 留痕审计

    5 年审计日志留存,支持监管现场检查,可导出标准格式报告。

    九、业务场景示例

    🛡 场景:分析师访问 L3 机密研报
    1. 用户(L3 clearance,投研部)提问:「查 ABC 新能源最新研报」
    2. 输入过滤:无敏感词,通过
    3. RBAC 检查:用户角色「投研分析师」含 read:knowledge,通过
    4. ABAC 检查:用户部门=投研部,文档 dept=投研部,匹配;用户 clearance=L3 ≥ 文档 L3,通过
    5. 数据级过滤:SQL/向量查询加 filter(dept_id=投研部 AND access_level<=L3)
    6. Agent 检索到研报,生成回答,引用研报
    7. 输出过滤:无敏感词,通过
    8. 审计日志:记录用户、时间、文档 ID、操作类型,签名链写入
    9. 若用户 clearance=L2,则在第 4 步被拒绝,审计记录拒绝原因
