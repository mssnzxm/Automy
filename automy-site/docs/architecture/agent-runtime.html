    • 智能体运行时(AgentScope Java 2.0 专题)

    智能体运行时是 Automy 的核心引擎,基于阿里达摩院开源的 AgentScope Java 2.0(Apache-2.0,JDK 17+,基于 Project Reactor 的响应式企业级 Agent 框架)构建。 本章节系统性地介绍 AgentScope 的设计哲学、核心组件、在 Automy 中的落地实践、与技能/专家层的关系、 模型网关对接、选型对比、局限对策,以及基于 OpenTelemetry 的执行追踪设计。这是技术架构中最核心的章节。

    ⚡
    AgentScope Java 2.0 一句话定义
    阿里达摩院开源的 Java 原生企业级 Agent 框架,ReActAgent 推理内核 + Harness 工程化底座 + Multi-Agent Supervisor 编排,MCP/A2A 原生,响应式高并发,与 Spring Boot 无缝融合。

    一、AgentScope Java 2.0 简介

    AgentScope Java 2.0 是阿里达摩院于 2026-07 正式 GA(Apache-2.0,v2.0.0)的企业级 Agent 框架,定位对标 LangGraph 等主流框架,但专为 Java 技术栈与生产级企业场景设计。其核心思想是「模型主导,框架退后」(model-led over fixed pipeline):当 LLM 推理足够强时,框架应退后让模型自主决策,而非用刚性 pipeline 约束 Agent 行为。

    AgentScope Java 2.0 运行于 JDK 17+,发布到 Maven Central,基于 Project Reactor 响应式流(非阻塞高并发),原生嵌入 Spring Boot 微服务体系。它将「推理内核」与「工程化底座」分层解耦:轻量场景用 ReActAgent,企业级长任务用 HarnessAgent 增强工作空间/会话/沙箱/可恢复能力。

    核心特性

    🧠 模型主导

    ReActAgent 推理内核轻量,让 LLM 自主规划工具调用,框架不约束固定 pipeline,适配推理模型快速演进。

    🏗 Harness 工程化

    HarnessAgent 在 ReActAgent 之上增加 Workspace/Session/Memory/Sandbox/Skill/Subagent,企业级有状态可恢复。

    🚀 响应式高并发

    基于 Project Reactor 响应式流,非阻塞 IO,单实例支撑高并发 Agent 调度与流式输出。

    🔗 MCP + A2A 原生

    原生支持 MCP(Model Context Protocol)与 A2A(Agent-to-Agent),可注册到 Nacos 跨进程跨机调用。

    🌱 Spring 生态融合

    原生嵌入 Spring Boot + Spring Cloud + Nacos,与团队既有 Java 微服务体系无缝对接。

    📊 OpenTelemetry 内置

    每一步 Reasoning step / Tool call 自动生成 Span,对接 LangFuse / 阿里云 ARMS,无需自接。

    开源生态

    • 官方仓库:github.com/agentscope-ai/agentscope-java
    • 官方文档:java.agentscope.io
    • 维护方:阿里达摩院 AgentScope 团队
    • 配套生态:Nacos(服务/Skill/MCP 市场管理)、Higress(AI 网关,模型代理 + MCP 代理)、QwenPaw、AgentTeams
    • 许可证:Apache-2.0,商用友好

    1. ReActAgent — 推理内核

    ReActAgent 是 AgentScope 的推理内核,采用 ReAct 循环(Reasoning + Tool Call)。它本身保持轻量,不绑定工程化状态,适合单轮/多步任务规划场景。Automy 用 ReActAgent 承担「意图识别 → 工具决策 → 推理生成」的轻量路径。

    // ReActAgent 示例:轻量推理 + 工具调用
import io.agentscope.core.*;
import io.agentscope.tool.*;

var searchTool = new SearchToolkit().build();
var codeTool   = new CodeToolkit().build();

var agent = ReActAgent.builder()
    .name("assistant")
    .model(new QwenConfig("qwen-plus"))
    .tools(List.of(searchTool, codeTool))
    .build();

// 响应式流式输出(Project Reactor)
Flux<String> stream = agent.stream("分析 A 公司的营收增长趋势");
stream.subscribe(System.out::print);

    2. HarnessAgent — 工程化底座

    HarnessAgent 在 ReActAgent 之上增加 Harness 工程化层(Workspace/Session/Memory/Sandbox/Skill/Subagent),面向企业级长任务:跨会话记忆、可恢复执行、权限审批、沙箱隔离。Automy 的「投研分析、尽调报告、法务审查」等长任务统一走 HarnessAgent。

    // HarnessAgent 示例:企业级长任务 + Redis Session + 沙箱
import io.agentscope.harness.*;

var harnessAgent = HarnessAgent.builder()
    .name("coder")
    .model(new QwenConfig("qwen-plus"))
    .filesystem(new LocalFilesystemSpec("./workspace"))
    .session(new RedisSession(jedisPool))
    .sandbox(SandboxMode.DOCKER)
    .build();

// 跨会话恢复:Redis Session + Workspace 快照
harnessAgent.resume(sessionId);

    3. Supervisor — 多 Agent 编排

    Supervisor 采用 Leader-Worker 模式编排多 Agent:Leader 负责任务分发与结果汇总,Worker(Subagent)负责领域执行。Subagent 通过 Markdown spec 声明,支持 同步阻塞 与 异步非阻塞 两种委派模式(DelegationMode.SYNC / ASYNC),替代传统 DAG 图编排。

    // Supervisor 多 Agent:投资分析(财务 + 团队评估)
var supervisor = Supervisor.builder()
    .name("investment-analyst")
    .subagents(List.of(
        SubagentSpec.fromMd("./subagents/financial-analyst.md"),
        SubagentSpec.fromMd("./subagents/team-evaluator.md")
    ))
    .mode(DelegationMode.ASYNC)  // 并行委派
    .build();

// 触发 Leader-Worker 协同
String report = supervisor.invoke("分析 A 公司,重点看财务和团队");

    4. Workspace / Memory / Session

    三大工程化组件共同支撑企业级有状态执行:

    • Workspace:基于 AbstractFilesystem 统一抽象(本地磁盘 → Docker 沙箱 → 对象存储,一行切换),workspace 路径按 sessionId/userId 隔离
    • Memory:MEMORY.md 自动管理 + 自动合并/压缩防上下文膨胀,对齐 Automy 三层记忆(事实/偏好/事件)
    • Session:基于 Redis 实现分布式 Session,跨副本会话恢复,支持零停机滚动发布

    5. Sandbox + Permission

    内置 Sandbox 执行技能代码与外部脚本,沙箱路径按 sessionId/userId 隔离,权限通过 Permission 系统控制,工具调用按需审批:

    ApprovalMode语义Automy 应用场景
    ALWAYS每次调用都需审批高风险:对外发邮件、删数据、调资金接口
    ONCE首次确认后本会话不再询问中风险:CRM 查询、Wind 数据拉取
    NEVER无需审批(已预授权)低风险:只读检索、向量召回

    6. @Tool 注解 + Toolkit

    AgentScope 用 @Tool 注解标记方法,框架自动生成 JSON Schema 注入提示词。Toolkit 是工具集合,可注册到 MCP 服务发现(Nacos)。Automy 通过此机制对接 MCP Tool Hub。

    // @Tool 注解 + Toolkit 示例
public class CrmToolkit {
    @Tool(desc = "查询客户商机列表")
    public List<Opportunity> listOpportunities(
        @Param("customerId") String customerId,
        @Param("limit") int limit
    ) {
        return crmClient.query(customerId, limit);
    }
}

// 注册到 Toolkit,自动生成 JSON Schema
var toolkit = Toolkit.of(new CrmToolkit());
agent = agent.toBuilder().toolkit(toolkit).build();

    技能 = AgentScope skills/*.md 声明式 Skill

    技能在 AgentScope 层面对应一个 声明式 Skill 文件(Markdown)。技能可独立开发、测试、发布,然后被多个专家 Subagent 复用。Subagent 通过 Markdown spec 嵌入技能列表。

    • 简单技能:单步工具调用,如「财务报表解析」= 加载 PDF → 提取表格 → 结构化输出
    • 复杂技能:多步 ReAct 循环,如「尽调报告」= 多源检索 + 多轮推理 + 报告生成
    • 技能组合:专家 Subagent spec 中引用多个技能文件,实现能力组合

    专家 = Supervisor + Subagent + 绑定资源

    专家是面向特定领域的高级 Agent,在 AgentScope 层面对应一个 Subagent 声明(Markdown spec),由 Supervisor 统一编排。

    七、局限与对策

    AgentScope Java 2.0 作为 2026-07 GA 的新框架,存在一些生态与能力上的局限,Automy 通过以下对策补齐:

    局限影响对策
    生态较新 2026-07 GA,社区文档以英文/阿里云为主,中文实战资料相对有限 内部沉淀最佳实践 + 模板 Subagent / Skill,关键路径文档化,留待实施期与官方文档(java.agentscope.io)校准
    Reactive 编程门槛 Project Reactor 响应式范式对团队是新概念,调试与排错心智成本高 路线图新增 M2.5「AgentScope Java 2.0 + Project Reactor 团队赋能」里程碑,关键路径文档 + Pair 编程
    部分企业系统无现成 Tool 适配 特定 CRM/ERP/数据中台无 AgentScope Toolkit 适配 通过 MCP Adapter 桥接已有企业 MCP Server,与既有 MCP 集成层设计一致
    RAG 高级模式 复杂 RAG 模式(HyDE、Multi-Query)需自行实现 在 knowledge/ + RAG 索引上自研 QueryRewriter、HyDE 生成器、多路召回融合器
    与既有 Spring 业务微服务边界 业务微服务与 Agent 运行时边界易混淆 架构文档明确:业务微服务保持 Spring Boot,Agent 编排独立部署 HarnessAgent 服务,通过 A2A/MCP 解耦

    九、工作流编排示例(Java)

    Java - 投资分析 Supervisor 多 Agent 工作流 
    // 投资分析 Supervisor:财务分析 + 团队评估 Subagent 并行委派
import io.agentscope.core.*;
import io.agentscope.harness.*;
import java.util.List;

public class InvestmentAnalystApp {

    public String analyze(String company) {
        // 1. 构建 Supervisor,加载两个 Subagent spec
        var supervisor = Supervisor.builder()
            .name("investment-analyst")
            .model(new QwenConfig("qwen-plus"))
            .subagents(List.of(
                SubagentSpec.fromMd("./subagents/financial-analyst.md"),
                SubagentSpec.fromMd("./subagents/team-evaluator.md")
            ))
            .mode(DelegationMode.ASYNC)  // 并行委派
            .filesystem(new MinioFilesystemSpec("workspace/investment"))
            .session(new RedisSession(jedisPool))
            .build();

        // 2. 异步非阻塞委派:财务 Subagent + 团队 Subagent 并行
        var reportFlux = supervisor.stream(
            "分析 " + company + ",重点看财务和团队"
        );

        // 3. 流式输出 + Trace 自动上报 LangFuse
        var report = new StringBuilder();
        reportFlux.doOnNext(report::append)
                  .subscribe(chunk -> System.out.print(chunk));

        return report.toString();
    }
}

    十、性能优化策略

    关键优化点
    • 响应式流式输出:Reactor Flux 端到端流式,降低 TTFT(首字节时间)< 800ms
    • 异步非阻塞委派:Supervisor DelegationMode.ASYNC 让多个 Subagent 并行执行
    • 工具缓存:相同输入的工具调用结果缓存(Redis,5 分钟 TTL)
    • 记忆预加载:会话开始时预加载 MEMORY.md,避免推理时阻塞
    • 上下文压缩:长会话自动合并/压缩 MEMORY.md,降低 Token 成本(超过 8K Token 触发压缩)
    • 模型分级:简单任务(意图识别、摘要)走小模型,复杂推理走大模型
    • MCP 连接池:MCP Client 连接池复用,避免每次调用重建连接
    • Workspace 快照:Redis Session + 对象存储快照,跨副本零停机滚动发布

    十一、执行流程总结

    1. 请求接收:Kong/Higress 网关接收用户请求,JWT 鉴权 + 限流
    2. Session 加载:Session Service 通过 Redis Session + Workspace 快照恢复完整上下文
    3. 专家加载:根据 expert_id 加载 Subagent Markdown spec + Persona + 资源绑定
    4. 上下文组装:加载 MEMORY.md、用户偏好、项目记忆
    5. ReAct 推理:ReActAgent 进入 ReAct 循环(Reasoning + Tool Call)
    6. 工具调用:@Tool + Toolkit 经 MCP Client 并行调用 MCP 工具,Permission 审批
    7. 模型推理:Model 抽象流式调用模型网关,Token 端到端回流
    8. 记忆写入:异步提取本次会话关键信息写入 MEMORY.md
    9. 结果返回:Reactor Flux 流式 SSE 输出到前端,标注引用与来源
    10. Trace 上报:全程 OpenTelemetry Span 上报 LangFuse / ARMS,审计服务消费落库
