    • 数据模型与存储

    Automy 采用「多存储引擎 + 混合持久化」架构,根据数据类型与访问模式选择最合适的存储。 本章节介绍存储分布、核心数据模型 ER 图、字段设计、分区策略、备份恢复与数据生命周期。 所有设计面向投资公司高合规要求(审计可追溯、防篡改、5 年留存)。

    🗄
    设计原则
    关系数据用 PG · 向量数据用 Milvus · 文件用对象存储 · 热数据缓存用 Redis · 审计日志独立存储防篡改

    三、核心表字段设计

    3.1 用户与组织

    PostgreSQL Schema - 用户/部门/角色 
    -- 用户表
CREATE TABLE users (
  id VARCHAR(36) PRIMARY KEY,
  username VARCHAR(64) UNIQUE NOT NULL,
  name VARCHAR(64) NOT NULL,
  email VARCHAR(128) UNIQUE,
  phone VARCHAR(20),
  avatar VARCHAR(256),
  dept_id VARCHAR(36) REFERENCES departments(id),
  title VARCHAR(64),
  clearance_level VARCHAR(8) DEFAULT 'L2',  -- 密级 L1-L4
  status VARCHAR(16) DEFAULT 'active',
  created_at TIMESTAMP DEFAULT NOW(),
  last_login TIMESTAMP
);

-- 部门表(树形结构)
CREATE TABLE departments (
  id VARCHAR(36) PRIMARY KEY,
  name VARCHAR(64) NOT NULL,
  parent_id VARCHAR(36) REFERENCES departments(id),
  path VARCHAR(256),  -- 如 /集团/投研部/TMT组
  sort INT DEFAULT 0,
  CHECK (id != parent_id)
);

-- 角色与权限(RBAC + ABAC)
CREATE TABLE roles (
  id VARCHAR(36) PRIMARY KEY,
  name VARCHAR(64) NOT NULL,
  permissions JSONB NOT NULL,  -- ["read:knowledge","call:mail.send"]
  is_builtin BOOLEAN DEFAULT FALSE
);

CREATE TABLE user_roles (
  user_id VARCHAR(36) REFERENCES users(id) ON DELETE CASCADE,
  role_id VARCHAR(36) REFERENCES roles(id),
  scope_type VARCHAR(16),  -- global/dept/project
  scope_id VARCHAR(36),
  PRIMARY KEY (user_id, role_id, scope_type, scope_id)
);

    3.2 会话与消息(分区表)

    -- 会话表
CREATE TABLE sessions (
  id VARCHAR(36) PRIMARY KEY,
  user_id VARCHAR(36) NOT NULL,
  project_id VARCHAR(36),
  expert_id VARCHAR(36) NOT NULL,
  title VARCHAR(128),
  status VARCHAR(16) DEFAULT 'active',
  pinned BOOLEAN DEFAULT FALSE,
  message_count INT DEFAULT 0,
  token_used BIGINT DEFAULT 0,
  created_at TIMESTAMP DEFAULT NOW(),
  updated_at TIMESTAMP
);
CREATE INDEX idx_sessions_user ON sessions(user_id, updated_at DESC);

-- 消息表(按月分区,支持海量历史消息)
CREATE TABLE messages (
  id VARCHAR(36) NOT NULL,
  session_id VARCHAR(36) NOT NULL,
  role VARCHAR(16) NOT NULL,  -- user/assistant/system/tool
  content TEXT NOT NULL,
  tokens INT,
  model VARCHAR(64),
  tool_calls JSONB,
  citations JSONB,
  trace_id VARCHAR(64),  -- OpenTelemetry Trace ID
  created_at TIMESTAMP NOT NULL,
  PRIMARY KEY (id, created_at)
) PARTITION BY RANGE (created_at);

-- 创建月度分区(自动维护)
CREATE TABLE messages_2026_07 PARTITION OF messages
  FOR VALUES FROM ('2026-07-01') TO ('2026-08-01');

    3.3 文档与分块(RAG 核心表)

    -- 文档表
CREATE TABLE documents (
  id VARCHAR(36) PRIMARY KEY,
  title VARCHAR(256) NOT NULL,
  type VARCHAR(16),  -- pdf/docx/xlsx/md
  category VARCHAR(64),
  size BIGINT,
  pages INT,
  sha256 VARCHAR(64) UNIQUE,  -- 内容哈希,去重
  storage_path VARCHAR(512),
  uploaded_by VARCHAR(36) REFERENCES users(id),
  project_id VARCHAR(36),
  dept_id VARCHAR(36),  -- 部门隔离
  access_level VARCHAR(8) DEFAULT 'L2',
  tags JSONB DEFAULT '[]',
  summary TEXT,
  status VARCHAR(16),  -- uploaded/parsing/parsed/failed
  parsed_at TIMESTAMP,
  created_at TIMESTAMP DEFAULT NOW()
);
CREATE INDEX idx_docs_project ON documents(project_id, access_level);
CREATE INDEX idx_docs_tags ON documents USING GIN(tags);

-- 文档分块表(用于 RAG,向量 ID 关联 Milvus)
CREATE TABLE document_chunks (
  id VARCHAR(36) PRIMARY KEY,
  document_id VARCHAR(36) NOT NULL REFERENCES documents(id) ON DELETE CASCADE,
  content TEXT NOT NULL,
  chunk_index INT NOT NULL,
  page INT,
  bbox JSONB,  -- 块在页面的位置
  vector_id VARCHAR(64),  -- Milvus 中的向量 ID
  heading_path VARCHAR(512),  -- H1/H2/H3 路径
  metadata JSONB
);
CREATE INDEX idx_chunks_doc ON document_chunks(document_id, chunk_index);

    3.4 审计日志(防篡改签名链)

    -- 审计日志表(独立库,只追加)
CREATE TABLE audit_logs (
  id BIGSERIAL PRIMARY KEY,
  user_id VARCHAR(36) NOT NULL,
  action VARCHAR(64) NOT NULL,  -- mcp.tool_call/skill.invoke/doc.read
  resource_type VARCHAR(32),
  resource_id VARCHAR(64),
  detail JSONB NOT NULL,
  ip VARCHAR(45),
  user_agent VARCHAR(256),
  trace_id VARCHAR(64),
  prev_hash CHAR(64) NOT NULL,  -- 上一条日志的 hash
  curr_hash CHAR(64) NOT NULL,  -- 本条 hash(含 prev_hash)
  created_at TIMESTAMP DEFAULT NOW()
);
-- 禁止 UPDATE/DELETE,通过规则实现
CREATE RULE no_update AS ON UPDATE TO audit_logs DO INSTEAD NOTHING;
CREATE RULE no_delete AS ON DELETE TO audit_logs DO INSTEAD NOTHING;

    六、备份与恢复

    备份策略(多层防护)
    • PostgreSQL:WAL 归档(连续)+ 每日全量备份,保留 30 天;PITR 支持,可恢复到任意秒
    • Milvus:每日快照,保留 7 天;binlog 持续归档
    • MinIO:多副本(至少 3 副本)+ 跨区域异步复制
    • Redis:RDB 每日 + AOF 实时持久化(appendfsync everysec)
    • 审计日志:独立备份,5 年留存(S3 WORM 不可篡改)
    • 恢复演练:每季度进行 DR 演练,RTO < 4 小时,RPO < 5 分钟

    七、数据安全与隐私

    🔒 静态加密

    • PG:透明数据加密(TDE),密钥存 KMS
    • Milvus:向量数据加密存储
    • MinIO:服务端加密 SSE-S3 / SSE-KMS
    • Redis:开启 TLS + at-rest 加密
    • 备份:加密后上传,密钥独立管理

    🔐 传输加密

    • 所有服务间通信强制 mTLS
    • 客户端到服务:TLS 1.3
    • 数据库连接:SSL 强制
    • 内部 API:Kong 网关统一 TLS 终止
    • 密钥轮换:90 天自动轮换
