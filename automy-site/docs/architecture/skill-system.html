    • 技能系统设计

    技能是 Automy 中可复用、可组合、可交易的能力单元,相当于 Agent 的「应用市场」。 技能系统负责技能的定义、存储、调度、执行、沙箱隔离与全生命周期管理。 本章节介绍技能数据模型、执行架构、沙箱设计、版本管理、市场治理与组合编排。

    ⚡
    设计目标
    能力复用避免重复造轮子 · 标准化输入输出 schema · 沙箱隔离保障安全 · 版本兼容平滑升级 · 市场化分发激励生态

    一、技能抽象模型

    每个技能是一段声明式工作流定义,包含输入/输出 schema、依赖工具、执行步骤、权限声明。 技能本身无状态,执行时由 Agent Runtime 注入上下文(用户、记忆、知识)。

    三、技能执行架构

    技能执行引擎基于 AgentScope HarnessAgent + Skill 文件,每个技能在执行时实例化为一个 HarnessAgent,在沙箱中隔离运行, 避免技能间相互污染或影响主服务。

    四、沙箱设计(安全隔离)

    技能可能由第三方开发者编写,直接在生产环境运行存在安全风险(恶意代码、资源滥用、数据泄露)。 Automy 采用 AgentScope 内置 Sandbox 实现毫秒级启动的强隔离沙箱(Docker + 对象存储快照)。

    沙箱隔离架构 Mermaid
    graph TB subgraph HOST["宿主机 Host"] MGR["Sandbox Manager
    调度 + 池化"] subgraph VM1["Sandbox 1(技能 A)"] A1["HarnessAgent"] A2["受限文件系统"] A3["网络白名单"] end subgraph VM2["Sandbox 2(技能 B)"] B1["HarnessAgent"] B2["受限文件系统"] B3["网络白名单"] end DOCKER["AgentScope 内置 Sandbox\n(Docker + 对象存储快照)"] end MGR --> VM1 MGR --> VM2 VM1 --> DOCKER VM2 --> DOCKER style VM1 fill:#FEF3C7,stroke:#D97706 style VM2 fill:#FCE7F3,stroke:#DB2777 style DOCKER fill:#E5E7EB,stroke:#6B7280
    隔离维度实现机制限制值
    计算隔离AgentScope 内置 Sandbox(Docker 容器隔离)CPU 2 核 / 内存 4Gi
    文件系统只读根文件系统 + 临时 overlay磁盘 10Gi,不可持久化
    网络隔离网络白名单(仅允许调用 MCP Gateway)禁止直连外网
    执行时长超时强杀默认 120s,最大 600s
    数据访问仅通过注入的上下文访问授权数据无法读取其他用户数据
    系统调用seccomp 过滤危险 syscall禁用 fork/exec/ioctl 等
    ⚠
    沙箱性能优化
    Docker 沙箱启动延迟 < 125ms,通过沙箱池预热的暖启动可降至 < 30ms;容器内存开销约 20MiB,单宿主机可并发 500+ 沙箱,快照持久化到对象存储(MinIO)跨副本可恢复。

    4.1 技能工具调用审批(AgentScope Permission)

    技能在沙箱内执行时,所有工具调用均经 AgentScope Permission 系统按需审批,防止技能越权操作敏感数据或外部服务。 审批模式由 ApprovalMode 配置,支持三种模式:

    ApprovalMode语义适用工具
    ALWAYS每次调用都需审批敏感工具:写操作、删除、网络外发、资金接口
    ONCE首次确认后本会话不再询问中风险工具:CRM 查询、Wind 数据拉取
    NEVER无需审批(已预授权)低风险工具:只读检索、向量召回

    默认策略:对敏感工具(写操作、删除、网络外发)默认 ApprovalMode.ALWAYS,确保每次执行都需用户或管理员确认。 人工审批流:审批请求经 AgentScope Event 系统发出事件,对接 Automy 既有审批流(飞书审批 / 工单系统),审批结果异步回灌后继续执行,审批全程留痕可审计。

    五、版本管理

    技能采用语义化版本(SemVer):MAJOR.MINOR.PATCH。 不同版本对用户安装的版本提供向后兼容承诺,平滑升级。

    六、技能组合编排

    技能可组合调用,形成更复杂的复合能力。复合技能本身也是技能,支持嵌套与并行。

    复合技能示例:端到端尽调报告(并行 + 串行) 
    name: due-diligence-pipeline
version: 1.0.0
category: 投研

steps:
  - id: step1
    skill: financial-report-parser      # 财报解析
    input: { document: "$input.report" }

  - id: step2
    skill: ipo-risk-analysis             # 风险分析
    input: { company: "$input.company" }
    depends_on: [step1]

  - id: step3
    skill: contract-review                # 合同审查(并行)
    input: { document: "$input.contracts" }
    parallel: true

  - id: step4
    action: generate-report               # 汇总报告
    input: ["$step1.output", "$step2.output", "$step3.output"]
    depends_on: [step2, step3]

    七、技能市场治理

    📋 上架审核

    • 代码扫描:静态分析恶意代码、后门
    • 权限审计:声明权限与实际使用一致
    • 内容合规:描述、图标无违规内容
    • 测试用例:必须提供 ≥5 个测试用例
    • 性能基线:平均执行 < 30s,P99 < 120s

    ⭐ 运营治理

    • 评分:1-5 星用户评分
    • 订阅量:展示安装数与活跃度
    • 下架规则:评分 < 3.0 自动下架
    • 收入分成:开发者 70% / 平台 30%
    • 版本支持:近 3 个版本并行支持

    八、技能 vs 专家 Agent

    维度技能 Skill专家 Agent
    定位原子能力单元(单一职责)完整角色(多技能 + 人格)
    有状态无状态有长期记忆
    调用方式函数式:invoke(input)→output对话式:与用户多轮交互
    组合关系被专家组合使用编排多个技能
    示例财报解析、合同审查、邮件起草投研分析师、法务顾问、合规官

    九、业务场景示例

    📈 场景:投研分析师调用「财报解析」技能
    1. 用户上传 ABC 新能源 2025 年财报,问「营收、毛利率、风险点」
    2. 投研分析师 Agent 决策调用 skill-financial-report-parser
    3. Skill Gateway 校验用户已订阅该技能(v2.1.0)
    4. 参数校验:input.document = 上传的 PDF
    5. 创建 AgentScope Sandbox,CPU 2 核 / 内存 4Gi / 超时 120s
    6. 沙箱内执行 HarnessAgent Skill:pdf.extract → knowledge.search(同业对比) → LLM 总结
    7. output_schema 校验通过,返回结构化结果(metrics + risk_points + summary)
    8. 沙箱销毁,资源回收;Agent 基于结果生成自然语言回答
