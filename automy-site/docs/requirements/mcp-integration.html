    • MCP 服务集成

    基于 Model Context Protocol 标准化协议,把企业 CRM、ERP、Wind、文档库、邮件等异构系统封装为 MCP 服务,用户主动连接、按需授权、全程审计,让 Agent 与企业系统的连接标准化、可控、可追溯。

    🔌
    模块定位
    本模块定义 MCP Adapter 框架、服务目录、用户主动连接、运行时管控(鉴权/脱敏/审计/限流/写操作确认)、离线部署五大能力,覆盖 MCP 服务从开发、注册、连接、调用到治理的全生命周期。

    模块概述

    投资公司内部存在大量异构企业系统:CRM、ERP、Wind、文档库、邮件、法律法规库等。传统集成方式每对接一个系统都要定制开发,成本高、周期长、难以复用,且权限边界混乱,审计困难。

    Automy 采用 Model Context Protocol(MCP) 标准协议,把企业系统统一封装为 MCP 服务,暴露 tools(工具调用)、resources(资源读取)、prompts(提示词模板)三类能力。用户主动勾选连接,按需授权范围,所有调用经过鉴权、脱敏、审计、限流、写操作确认五重管控,既保证 Agent 能力扩展性,又确保数据主权与合规边界。

    需求条目

    REQ-6.1 MCP Adapter 框架
    P0

    系统 SHALL 提供 MCP Adapter 框架,把企业系统封装为标准 MCP 服务,暴露 tools(工具调用)、resources(资源读取)、prompts(提示词模板)三类能力;SHALL 提供 TypeScript / Python SDK 简化 Adapter 开发;SHALL 支持自定义 Adapter 接入。

    • tools:可被 Agent 调用的工具函数,如 crm.query_customer、wind.get_quote
    • resources:可被 Agent 读取的资源,如 doc://research/12345
    • prompts:可复用的提示词模板,如 prompt://contract-review-template
    • SDK:TypeScript / Python SDK,声明式定义工具与资源
    • 自定义 Adapter:支持开发自定义 MCP 服务接入,通过市场发布共享
    • 协议规范:遵循 MCP 协议规范,支持 JSON-RPC 2.0 通信

    场景

    WHEN 投研团队想把 Wind 金融数据系统接入 Automy
    THEN 团队使用 Python SDK 开发 Wind MCP Adapter,定义 get_realtime_quote(实时行情)、get_financial_data(历史财务)、get_macro_indicator(宏观指标)三个工具,注册到 Automy 服务目录,全员可用。

    REQ-6.2 服务目录
    P0

    系统 SHALL 提供统一的 MCP 服务目录,展示所有可用 MCP 服务的清单、能力说明、所需权限、数据范围、认证方式;SHALL 支持分类浏览、关键词搜索;管理员 SHALL 可发布新服务到目录。

    • 服务清单:展示服务名称、提供方、分类、能力说明、订阅数
    • 能力详情:展开查看服务暴露的 tools / resources / prompts 清单与参数
    • 权限说明:展示服务所需的权限(读 CRM / 写邮件 / 读财务数据等)
    • 数据范围:展示服务可访问的数据范围(本部门 / 全公司 / 跨公司)
    • 认证方式:展示服务所需的认证方式(OAuth 2.0 / API Key / SSO)
    • 服务发布:管理员可在后台发布新 MCP 服务到目录,设置可见范围

    场景

    WHEN 张三在 MCP 服务目录浏览
    THEN 张三看到 7 个可用服务,点击 "Wind 金融数据" 展开详情:能力(实时行情 / 历史财务 / 宏观指标)、所需权限(读 Wind 数据)、认证方式(API Key)、订阅数 156;张三可点击 "连接" 进入授权流程。

    REQ-6.3 用户主动连接
    P0

    系统 SHALL 坚持用户主动连接原则,绝不默认连接任何企业系统;用户勾选连接后 SHALL 弹窗展示授权范围,用户逐项授权;SHALL 支持随时撤销连接,撤销后立即停止所有调用。

    • 主动连接:用户在 MCP 服务目录勾选要连接的服务,系统不自动连接
    • 授权范围:连接时弹窗展示授权清单(如 "读 CRM 客户列表" / "写邮件" / "读财务数据"),用户逐项授权
    • 最小权限:默认仅授予读权限,写权限需单独勾选
    • 授权撤销:用户可随时在 "我的连接" 撤销授权,撤销后立即停止所有调用
    • 连接状态:实时显示连接状态(已连接 / 已断开 / 授权过期)与最近调用时间
    • 授权审计:所有授权与撤销操作记录到审计日志

    场景

    WHEN 张三连接 "CRM 系统" MCP 服务
    THEN 系统弹窗展示授权清单:✅ 读客户列表(默认勾选) / ✅ 读商机详情(默认勾选) / ⬜ 写客户信息(默认未勾选) / ⬜ 删除客户(默认未勾选);张三按需勾选后点击 "授权",完成 OAuth 2.0 认证,CRM 服务进入 "已连接" 状态。
    WHEN 张三在 "我的连接" 撤销 "CRM 系统" 授权
    THEN 系统立即停止所有 CRM 工具调用,正在进行的调用被中断;后续 Agent 调用 CRM 工具时返回 "未授权" 错误,提示张三重新连接;撤销操作记录到审计日志。

    REQ-6.4 运行时管控
    P0

    系统 SHALL 对所有 MCP 工具调用实施运行时管控,包括鉴权、脱敏、审计、限流、写操作确认五重管控;SHALL 确保任何调用不可绕过管控链路。

    管控环节说明示例
    鉴权调用前校验用户是否有该工具权限张三调用 crm.query_customer 前校验其是否已授权 CRM 读权限
    脱敏返回数据按规则脱敏客户手机号 138****1234,身份证 110***********1234
    审计记录调用者、时间、参数、结果审计日志:张三 2026-07-15 10:23 调用 wind.get_quote(ABC) 返回 ¥50.2
    限流按用户与服务限流,防止滥用Wind 数据服务:单用户 60 次/分钟,全平台 600 次/分钟
    写操作确认写操作需用户二次确认crm.update_customer 前弹窗展示将修改的字段与值,用户确认后执行
    • 管控链路:鉴权 → 限流 → 调用 → 脱敏 → 审计,任何环节失败均中断调用
    • 写操作清单:管理员维护写操作清单(如 update / delete / send),清单内操作需确认
    • 敏感数据脱敏:按规则脱敏手机号、身份证、银行卡、金额(超阈值)
    • 审计字段:operator / service / tool / params / result / timestamp / ip / sessionId
    • 限流策略:支持按用户、按服务、按工具维度限流,超限返回 429

    场景

    WHEN 张三的 Agent 调用 mcp.crm.query_customer 查询客户信息
    THEN 系统依次执行:(1) 鉴权:张三已授权 CRM 读权限 ✅;(2) 限流:本分钟已调用 5 次,未超限 ✅;(3) 调用 CRM 返回客户数据;(4) 脱敏:客户手机号 138****1234,身份证 110***********1234;(5) 审计:记录完整调用日志。
    WHEN Agent 调用 mcp.email.send 发送邮件
    THEN 系统识别为写操作,弹窗展示 "即将发送邮件:收件人 zhangsan@corp.com,主题 ABC 项目进展,正文摘要...";张三点击 "确认发送" 后执行,审计日志记录邮件内容摘要。

    REQ-6.5 离线部署
    P1

    系统 SHALL 支持内网纯离线部署场景,所有 MCP 服务 Adapter 可内网部署,不依赖公网;SHALL 提供内网 MCP 服务市场,支持内网共享自定义 Adapter。

    • 内网部署:MCP Adapter 可打包为 Docker 镜像内网部署,无需公网访问
    • 内网市场:提供内网 MCP 服务市场,企业内自研 Adapter 可在内网共享
    • 认证适配:支持企业内网 SSO / LDAP 认证,不依赖公网 OAuth
    • 数据驻留:所有 MCP 调用数据留在内网,不外发
    • 离线文档:提供完整的离线部署文档与依赖清单

    场景

    WHEN 投资公司要求所有系统纯内网部署,不接入公网
    THEN Automy 与所有 MCP Adapter 全部内网部署,CRM / ERP / Wind(内网代理)/ 邮件系统通过内网调用;自研的 "投资管理系统 MCP Adapter" 发布到内网市场,全员可用;数据不出企业边界。
