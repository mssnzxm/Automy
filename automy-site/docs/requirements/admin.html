    • 平台管理后台

    面向平台管理员的运维与配置界面。提供用户与组织管理、角色权限管理、审计日志查看、模型配置、市场审核、系统配置(KMS、脱敏规则、数据驻留)六大能力,是运维人员的核心工作界面。

    ⚙
    模块定位
    本模块定义管理员视角的全部能力:用户与组织管理(REQ-10.1)、角色权限管理(REQ-10.2)、审计日志查看(REQ-10.3)、模型配置(REQ-10.4)、市场审核(REQ-10.5)、系统配置(REQ-10.6)。

    模块概述

    Automy 作为企业级平台,需要管理员进行日常运维:用户管理、模型路由配置、市场内容审核、性能监控与告警。管理后台是运维人员、IT 管理员、合规审计员的核心工作界面,需要提供清晰的可视化与高效的批量操作能力。

    管理后台遵循最小权限与职责分离原则:普通管理员负责用户与配置,审计员专注合规审计,超级管理员统筹全局。所有管理操作均记录到审计日志,可追溯。

    需求条目

    REQ-10.1 用户与组织管理
    P0

    系统 SHALL 提供用户与组织管理界面,支持用户列表查看、搜索、筛选、批量操作;SHALL 支持维护部门树、团队、项目组;SHALL 支持账号启停、密码重置、角色分配。

    • 用户列表:展示所有用户,支持按姓名、部门、角色、状态搜索与筛选
    • 批量操作:批量启用/禁用、批量分配角色、批量导入(Excel)
    • 组织架构:可视化维护部门树、团队、项目组,支持拖拽调整
    • 账号管理:启用/禁用/锁定用户账号,重置密码,强制下线
    • 角色分配:为用户分配一个或多个角色,支持角色继承
    • 用户详情:查看用户基本信息、登录历史、权限、订阅、记忆元数据

    场景

    WHEN 管理员需要批量导入新员工账号
    THEN 管理员在用户管理页面上传 Excel(包含姓名、邮箱、部门、工号),系统校验数据后批量创建账号,自动同步组织信息,分配默认 employee 角色,通过邮件发送初始密码;导入结果报告展示成功 50 条、失败 2 条(邮箱重复)。

    REQ-10.2 角色权限管理
    P0

    系统 SHALL 提供角色权限管理界面,支持查看预置角色、创建自定义角色、配置权限点;SHALL 支持角色继承与权限矩阵可视化;SHALL 对所有权限变更记录审计。

    • 预置角色:查看 employee / skill-author / admin / auditor 四类预置角色权限
    • 自定义角色:基于预置角色派生自定义角色,绑定权限点
    • 权限点管理:查看所有权限点(按模块分类:工作台 / 技能 / 专家 / 知识库 / MCP / 记忆 / 审计)
    • 权限矩阵:可视化展示角色与权限点的对应关系
    • 角色继承:支持角色层级,子角色继承父角色权限
    • 权限变更审计:所有角色创建、修改、删除、用户角色分配变更记录到审计日志

    场景

    WHEN 管理员创建 "投研部高级分析师" 自定义角色
    THEN 管理员基于 employee 角色派生,添加权限点:create_skill / publish_skill_to_team / access_research_kb;保存后该角色出现在角色列表,可分配给投研部骨干;变更记录到审计日志。

    REQ-10.3 审计日志查看
    P0

    系统 SHALL 提供审计日志查看界面,支持按用户、部门、时间范围、操作类型、资源类型多维检索;SHALL 支持查看日志详情、导出合规报告;SHALL 提供异常行为告警。

    • 多维度检索:按用户、部门、时间范围、操作类型、资源类型、IP 等检索
    • 日志详情:点击日志查看完整详情,包含参数、结果、上下文
    • 合规报告:导出指定时间范围的审计报告(PDF / Excel),支持自定义字段
    • 异常告警:检测异常行为(短时间大量删除、越权访问、敏感数据访问)并实时告警
    • 日志保留:审计日志保留 1 年,敏感操作保留 3 年
    • 日志防篡改:展示日志哈希链,验证日志完整性

    场景

    WHEN 审计员李四需要调查某员工的异常数据访问
    THEN 李四在审计日志页面按用户筛选,选择时间范围最近 7 天,操作类型选 "敏感数据访问";系统返回 12 条记录,李四点击详情查看每次访问的具体资源与参数,确认无异常后导出 Excel 报告归档。

    REQ-10.4 模型配置
    P0

    系统 SHALL 提供模型配置界面,支持添加、编辑、删除模型;SHALL 配置路由策略、限流策略、配额;SHALL 提供模型调用监控仪表盘与成本统计报表。

    • 模型列表:展示所有已配置模型,含名称、类型(自部署/商用)、endpoint、状态
    • 新增模型:填写 endpoint、模型名、API Key、能力声明(上下文长度、视觉、函数调用)
    • 路由策略:配置按场景、负载、成本、敏感度的路由规则
    • 限流策略:按用户、部门、模型维度配置限流
    • 配额管理:为用户/部门设置月度 Token 配额
    • 成本统计:按用户、部门、模型、场景统计 Token 用量与成本,导出报表
    • 监控仪表盘:实时展示调用量、响应时延、错误率、成本趋势
    • 降级配置:配置主备模型与降级规则

    场景

    WHEN 管理员想查看本月模型调用成本
    THEN 管理员在模型配置 → 成本统计,选择本月时间范围,看到按部门统计的成本:投研部 ¥12,345(占比 45%)、法务部 ¥5,678(占比 21%)、合规部 ¥3,456(占比 13%);可下钻查看明细,导出 Excel 报表。

    REQ-10.5 市场审核
    P1

    系统 SHALL 提供市场审核界面,管理员可审核技能与专家发布申请;SHALL 展示安全扫描结果、权限审查结果;SHALL 支持通过、驳回、待修改、紧急下架操作。

    • 审核队列:展示待审核的技能与专家发布申请,按提交时间排序
    • 审核详情:展示技能定义、安全扫描结果、权限审查结果、作者信息
    • 审核操作:通过 / 驳回(附理由)/ 待修改(附建议)/ 紧急下架
    • 批量审核:支持批量通过或驳回
    • 申诉处理:处理作者对驳回决定的申诉
    • 审核日志:所有审核操作记录到审计日志

    场景

    WHEN 管理员收到 5 个技能发布申请
    THEN 管理员在市场审核页面查看队列,逐个审核:(1) "财务报表解析 v2.1" 安全扫描通过、权限合理 → 通过;(2) "客户洞察分析" 检测到提示词注入风险 → 驳回(附理由);(3) "会议纪要模板" 缺少使用示例 → 待修改(附建议)。

    REQ-10.6 系统配置(KMS、脱敏规则、数据驻留)
    P1

    系统 SHALL 提供系统配置界面,涵盖 KMS 密钥管理、脱敏规则配置、数据驻留策略、SSO 配置、邮件配置、存储配置、安全配置;所有配置变更 SHALL 记录审计并支持回滚。

    配置项说明示例
    KMS 密钥管理加密密钥配置与轮换主密钥每 90 天轮换,数据密钥每 30 天轮换
    脱敏规则敏感字段脱敏规则配置手机号 138****1234,身份证 110***********1234
    数据驻留数据物理边界配置客户持仓数据仅在上海机房
    SSO 配置企业 SSO 对接配置OIDC endpoint / client_id / client_secret
    邮件配置SMTP 服务器与通知模板SMTP: smtp.corp.com,模板:审核通知
    存储配置文件存储与数据库连接S3 endpoint / 数据库连接池
    安全配置密码策略、IP 白名单、敏感词密码 12 位+大小写+数字,IP 白名单 10.0.0.0/8
    • 配置审计:所有配置变更记录到审计日志,含变更前后值
    • 配置回滚:支持配置回滚到历史版本
    • 配置校验:保存前校验配置有效性(如 SMTP 连通性、SSO endpoint 可达性)
    • 敏感配置加密:client_secret、API Key 等敏感配置加密存储,展示时脱敏
    • 配置导入导出:支持配置导出为 JSON,便于环境迁移

    场景

    WHEN 管理员配置 KMS 密钥轮换策略
    THEN 管理员在系统配置 → KMS,设置主密钥轮换周期 90 天、数据密钥轮换周期 30 天;保存后系统按周期自动轮换密钥,旧密钥保留用于解密历史数据,新数据使用新密钥加密;变更记录到审计日志。
    WHEN 管理员修改 SSO 配置填错 client_id
    THEN 系统保存前自动校验 OIDC endpoint 可达性与 client_id 有效性,校验失败提示 "OIDC 配置无效,请检查 client_id";管理员修正后保存成功,变更记录到审计日志。
