    • 集成与扩展

    开放 Automy 能力给外部系统调用,同时支持将外部能力集成到 Automy 工作流。包括模型网关(多模型路由、限流、配额、成本统计)、多模型支持(自部署 + 商用 API)、第三方工具扩展、Webhook 与事件四大能力。

    🔗
    模块定位
    本模块定义 Automy 与外部世界的集成边界:对内统一管理多模型路由与成本,对外提供 Open API、SDK、Webhook、嵌入式组件,让 Automy 能力可被嵌入到投资管理系统、OA、BI 等现有系统中。

    模块概述

    投资公司已有大量内部业务系统(投资管理系统、风控系统、OA 系统、BI 系统等),希望在不替换现有系统的前提下,把 Automy 的 AI 能力嵌入到员工日常工作流中。同时,投资公司对模型选型有差异化需求:涉及敏感数据的场景必须用本地部署模型,通用场景可用商用 API 降本。

    本模块解决两大核心问题:(1)模型网关统一管理多模型路由、限流、配额、成本统计,支持自部署 Qwen / DeepSeek / GLM 与商用 API 混合使用;(2)开放集成提供 Open API、SDK、Webhook、嵌入式组件,让 Automy 能力可被嵌入到任何现有系统中。

    需求条目

    REQ-8.1 模型网关
    P0

    系统 SHALL 提供统一的模型网关,实现多模型路由、限流、配额管理、成本统计、降级容错;SHALL 支持按场景、负载、成本、敏感度四种路由策略;SHALL 提供模型调用监控仪表盘。

    路由策略说明示例
    按场景不同场景路由到不同模型财报解析 → DeepSeek(数学强);合同审查 → GLM(中文强)
    按负载负载均衡,避免单模型过载Qwen-72B 队列长度 > 10 时路由到备用 GLM-4
    按成本优先使用低成本模型,失败再升级通用问答优先用 Qwen-7B,失败再升级到 Qwen-72B
    按敏感度敏感数据强制路由到本地模型涉及客户信息的对话强制使用本地 GLM,不走商用 API
    • 限流:按用户、按部门、按模型维度限流,防止滥用
    • 配额管理:为用户/部门设置月度 Token 配额,超配额降级或拒绝
    • 成本统计:按用户、部门、模型、场景统计 Token 用量与成本,导出报表
    • 降级容错:主模型不可用时自动降级到备用模型,记录降级事件
    • 监控仪表盘:实时展示调用量、响应时延、错误率、成本趋势
    • 缓存优化:相同请求缓存结果,减少重复调用成本

    场景

    WHEN 张三与专家讨论涉及某客户的投资策略
    THEN 模型网关检测到对话涉及客户敏感信息,按敏感度策略路由到本地部署的 GLM-4 模型,不走商用 API;调用成本计入投研部账户,Token 用量实时扣减配额。
    WHEN 主模型 DeepSeek-V3 服务不可用
    THEN 网关自动降级到备用模型 GLM-4,用户无感知;运维收到告警 "DeepSeek-V3 不可用,已降级到 GLM-4",降级事件记录到审计日志;DeepSeek 恢复后自动切回。

    REQ-8.2 多模型支持
    P0

    系统 SHALL 同时支持自部署开源模型与商用 API 模型;自部署模型 SHALL 至少支持 Qwen、DeepSeek、GLM、Llama 系列;商用 API SHALL 至少支持 OpenAI、Anthropic Claude、阿里通义、智谱 GLM;SHALL 支持模型热插拔,新增模型无需重启。

    类型模型典型场景部署方式
    自部署Qwen-72B / Qwen-7B通用问答、文档摘要vLLM / TGI 内网部署
    DeepSeek-V3 / DeepSeek-R1数学推理、财报分析vLLM 内网部署
    GLM-4 / GLM-4V中文场景、多模态智谱官方部署 / 内网
    商用 APIOpenAI GPT-4 / GPT-4o复杂推理、多模态Azure OpenAI / 直连
    Anthropic Claude 3.5长文档分析、代码API 直连
    通义千问 / 智谱 GLM中文优化、合规阿里云 / 智谱 API
    • 统一接口:所有模型通过统一 OpenAI 兼容 API 接入,屏蔽差异
    • 能力声明:每个模型声明能力(上下文长度、是否支持视觉、是否支持函数调用)
    • 热插拔:新增模型只需在管理后台配置 endpoint 与 API Key,无需重启
    • 模型对比:支持同一问题用不同模型生成,并排对比效果
    • Embedding 模型:支持 BGE、text-embedding-3、智谱 embedding 用于向量化

    场景

    WHEN 管理员新增一个本地部署的 DeepSeek-V3 模型
    THEN 管理员在管理后台 → 模型配置 → 新增模型,填写 endpoint(http://gpu-01:8000/v1)、模型名(deepseek-v3)、API Key(空)、能力(上下文 128K,支持函数调用);保存后模型立即可用,无需重启服务。

    REQ-8.3 第三方工具扩展
    P1

    系统 SHALL 提供 Open API 与 SDK,支持第三方系统调用 Automy 的会话、技能、专家、知识检索能力;SHALL 提供嵌入式组件,可嵌入到第三方系统;SHALL 支持第三方开发自定义 MCP 服务接入。

    • Open API:RESTful API,支持会话创建、消息发送、技能调用、专家调用、知识检索
    • 流式响应:API 支持 SSE 流式输出,与工作台体验一致
    • 认证授权:基于 API Token 的认证,Token 绑定权限范围
    • 限流:按 Token 限流,防止滥用
    • SDK:提供 Python / JavaScript SDK,简化集成开发
    • 嵌入式组件:可嵌入到第三方系统的对话浮窗、技能调用按钮、知识检索框
    • SSO 集成:嵌入式组件与第三方系统共享登录态,无需重复登录

    API 调用示例

    Python 
    from automy import Automy

client = Automy(
    endpoint="https://automy.internal.corp",
    token="your-api-token"
)

# 创建会话并调用专家
session = client.create_session(expert="investment-analyst")

for chunk in session.stream("分析 ABC 新能源 2025 年财报"):
    print(chunk.content, end="", flush=True)

    REQ-8.4 Webhook 与事件
    P1

    系统 SHALL 提供 Webhook 机制,支持订阅 Automy 的事件(专家完成、记忆写入、审计事件、技能审核结果);SHALL 在事件发生时回调通知第三方系统;SHALL 携带签名防止伪造。

    • 事件类型:session.completed(会话完成)、memory.written(记忆写入)、audit.event(审计事件)、skill.approved(技能审核通过)、expert.subscribed(专家订阅)
    • 订阅管理:用户/管理员可订阅感兴趣的事件类型,配置回调 URL
    • 回调通知:事件发生后异步回调,支持重试(最多 3 次,指数退避)
    • 签名校验:Webhook 携带 HMAC-SHA256 签名,防止伪造
    • 事件负载:包含事件类型、时间戳、关联资源 ID、操作者、变更摘要
    • 事件日志:记录所有 Webhook 调用日志,可重放失败的事件

    场景

    WHEN 投资公司 OA 系统需要在 Automy 技能审核通过后自动通知作者
    THEN OA 系统订阅 skill.approved 事件,配置回调 URL;当管理员审核通过 "财务报表解析" 技能时,Automy 异步回调 OA 系统 Webhook,携带事件负载 {event: skill.approved, skillId: fin-report-parser, author: zhangsan};OA 系统收到后向张三发送企业微信通知。
    WHEN Webhook 回调失败(第三方系统不可用)
    THEN Automy 重试 3 次(间隔 1 分钟 / 5 分钟 / 30 分钟,指数退避);3 次均失败后标记为 "delivery_failed",管理员可在 Webhook 日志中查看并手动重放。

    集成场景

    投资管理系统嵌入

    将 Automy 对话浮窗嵌入到现有投资管理系统中,员工不切换系统即可使用 AI 能力。

    OA 流程自动化

    通过 Webhook 在 OA 审批流程中调用 Automy 技能,自动化文档生成与审核。

    BI 系统增强

    在 BI 报表中通过 API 调用 Automy 生成自然语言解读与洞察。

    移动端集成

    通过 SDK 在企业移动 App 中集成 Automy 能力。

    • ✅ 多模型支持完整,自部署与商用 API 混合使用
    • ✅ Open API 与 SDK 易用,嵌入式组件可集成
    • ✅ Webhook 事件订阅与回调正常,签名校验有效
    • ✅ 模型降级容错,用户无感知
