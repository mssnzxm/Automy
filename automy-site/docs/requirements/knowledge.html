    • 知识库与 RAG

    企业内部知识沉淀、检索与复用。基于 RAG 检索增强生成,让 Agent 能精准引用企业知识库中的具体文档与数据,所有回答附带文件名与页码,可跳转原文验证,告别 AI 幻觉。

    📚
    模块定位
    本模块定义文档接入(多源)、文档处理流水线(解析/分块/向量化/元数据)、混合检索(语义+关键词+元数据)、来源引用(文件名+页码)、未命中兜底五大能力,让企业知识资产可被 Agent 精准引用。

    模块概述

    投资公司积累了大量研究报告、行业研究、尽调材料、内部备忘录、法规案例等知识资产。这些资产分散在各系统(文档库、邮件、研究系统、网页),格式多样(PDF、Word、Excel、PPT、HTML),难以检索与复用。

    Automy 知识库通过 RAG(Retrieval-Augmented Generation)技术,让 Agent 能精准引用企业内部知识:支持多源接入(本地 / 文档系统 / 邮件 / 网页),自动完成解析、分块、向量化、元数据提取的文档处理流水线,基于语义+关键词+元数据的混合检索提升召回率,所有回答附带文件名与页码,可跳转原文验证,确保 AI 回答可追溯、可信。

    知识库分类

    分类典型内容访问范围
    公司知识库研究报告、行业研究、宏观分析全公司
    部门知识库部门内部资料、流程文档部门内
    项目知识库尽调材料、访谈记录、纪要项目成员
    个人知识库个人笔记、收藏、草稿个人

    需求条目

    REQ-7.1 文档接入
    P0

    系统 SHALL 支持多源文档接入,包括本地文件上传、文档系统对接(SharePoint / 飞书文档 / 腾讯文档)、邮件接入、网页爬取;SHALL 支持多格式文件(PDF、Word、Excel、PPT、Markdown、HTML)。

    • 本地上传:支持单文件与批量上传,单文件 ≤ 100MB,批量 ≤ 50 文件
    • 文档系统:对接 SharePoint / 飞书文档 / 腾讯文档,定时增量同步
    • 邮件接入:从邮箱自动提取附件与正文,按规则归档到知识库
    • 网页爬取:配置 URL 与爬取规则,定时抓取公开网页(如监管公告)
    • 格式支持:PDF / Word / Excel / PPT / Markdown / HTML / TXT
    • 权限继承:文档接入时继承源系统的权限,如 SharePoint 文档的部门权限

    场景

    WHEN 投研团队把 SharePoint 上的 "2025 年度研究报告" 文件夹接入 Automy 知识库
    THEN 系统定时增量同步该文件夹,所有 PDF / Word 报告自动入库,继承 SharePoint 的部门权限(仅投研部可访问),新文档发布后 1 小时内同步到 Automy 可检索。

    REQ-7.2 文档处理流水线
    P0

    系统 SHALL 对接入的文档自动执行解析、分块、向量化、元数据提取的处理流水线;SHALL 支持表格、图片、公式等多模态内容提取;处理状态与进度对用户可见。

    环节说明示例
    解析从源文件提取文本、表格、图片、公式PDF 解析提取正文 + 表格 + 图表说明
    分块按语义与结构切分为可检索的块按段落分块,每块 500-1000 字,保留上下文
    向量化生成嵌入向量,存入向量库使用 BGE / text-embedding-3 生成向量
    元数据提取文档元数据(标题、作者、时间、分类)标题:ABC 新能源 2024 年报,作者:投研部,时间:2025-03-15
    索引建立全文索引与向量索引Elasticsearch 全文 + Milvus 向量
    • 多模态提取:支持表格(转为结构化数据)、图片(OCR + 描述)、公式(LaTeX)
    • 分块策略:按段落、按章节、按页码分块,保留上下文与位置信息
    • 增量更新:文档更新时增量重建索引,不全量重建
    • 处理状态:展示处理进度(待处理 / 处理中 / 已完成 / 失败),失败可重试
    • 处理日志:记录每步处理结果,便于排查问题

    场景

    WHEN 张三上传一份 50 页的 ABC 新能源年报 PDF
    THEN 系统自动执行:解析(提取正文 + 12 个表格 + 8 张图表)→ 分块(按章节切为 80 个块)→ 向量化(生成 80 个向量)→ 元数据(标题:ABC 新能源 2024 年报,作者:ABC 公司,时间:2025-03-15);处理状态从 "处理中" 变为 "已完成",全程耗时约 30 秒。

    REQ-7.3 混合检索
    P0

    系统 SHALL 提供混合检索能力,综合语义检索(向量相似度)、关键词检索(BM25)、元数据过滤(部门、时间、分类)三种策略;SHALL 对召回结果进行重排序,提升精度;检索时延 SHALL ≤ 2 秒。

    • 语义检索:基于向量相似度,理解语义而非字面匹配,如 "新能源车" 命中 "电动汽车"
    • 关键词检索:基于 BM25 算法,精确匹配专业术语、人名、公司名
    • 元数据过滤:按部门、时间范围、分类、密级过滤,如 "仅投研部 2025 年的研究报告"
    • 重排序:使用交叉编码器对召回结果重排序,提升 Top-K 精度
    • 检索时延:P95 ≤ 2 秒,Top-10 召回
    • 权限过滤:检索时自动过滤用户无权访问的文档

    场景

    WHEN 张三提问 "ABC 新能源动力电池业务 2024 年表现如何"
    THEN 系统执行混合检索:(1) 语义检索召回 20 条相关块;(2) 关键词检索补充命中 "ABC 新能源" 与 "动力电池" 的块;(3) 元数据过滤(投研部 / 2024-2025 年);(4) 重排序后返回 Top-5 块,时延 1.3 秒。

    REQ-7.4 来源引用
    P0

    系统 SHALL 在 Agent 回答中标注引用的源文档文件名与页码,SHALL 支持点击跳转原文位置;SHALL 在引用来源区展示所有引用的文档清单,便于用户验证。

    • 行内引用:回答中关键信息后标注 [1] [2] 等引用编号
    • 引用来源区:消息底部展示引用清单,每条引用显示文件名、页码、片段预览
    • 跳转原文:点击引用编号可跳转到原文档的具体页码与位置
    • 片段预览:鼠标悬停引用编号展示原文片段(上下文)
    • 引用可信度:展示每条引用的相关度评分
    • 无引用标注:Agent 回答若未基于知识库,SHALL 明确提示 "此回答未基于知识库,仅供参考"

    场景

    WHEN 张三问 "ABC 新能源 2024 年动力电池业务收入是多少"
    THEN 专家回答:"ABC 新能源 2024 年动力电池业务收入 ¥32.5 亿,同比增长 45% [1]。其中磷酸铁锂电池占比 70%,三元锂电池占比 30% [2]。";引用来源区展示:[1] ABC 新能源 2024 年年报.pdf 第 23 页;[2] ABC 新能源 2024 年年报.pdf 第 24 页;张三点击 [1] 跳转到 PDF 第 23 页对应位置。

    REQ-7.5 未命中兜底
    P1

    系统 SHALL 在知识库未命中或命中相关度低时,提供兜底策略,包括:明确告知用户未命中、降级使用模型通用知识(明确标注)、推荐相关文档供用户参考、提供 "提交问题到知识库管理员" 入口。

    • 未命中提示:检索 Top-K 结果相关度均低于阈值(默认 0.6)时,明确告知 "知识库中未找到高度相关内容"
    • 降级回答:可配置降级为模型通用知识回答,但 SHALL 明确标注 "以下回答未基于知识库,仅供参考"
    • 相关推荐:展示 Top-3 相关度较低的文档,供用户参考
    • 问题提交:提供 "提交此问题到知识库管理员" 入口,管理员可补充文档
    • 未命中统计:统计未命中率,辅助管理员发现知识库覆盖盲区

    场景

    WHEN 张三问 "DEF 公司 2024 年海外业务占比" 但知识库中没有 DEF 公司相关文档
    THEN 专家回答:"知识库中未找到 DEF 公司相关内容。以下基于通用知识回答:DEF 公司是一家... (此回答未基于知识库,仅供参考)。推荐查看相关文档:[1] 行业龙头海外业务对比.pdf(相关度 0.42)。如需更准确回答,可提交此问题到知识库管理员。"
