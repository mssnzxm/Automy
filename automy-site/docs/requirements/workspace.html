    • 工作台与会话

    用户与 Agent 交互的主入口。支持多会话并行管理、SSE/WebSocket 流式输出、工具调用过程透明展示、随时中断与追问、按项目组织会话与文档,让投资公司员工高效处理多项目并行工作。

    💼
    模块定位
    本模块是用户体验层的核心,定义会话生命周期管理、流式输出协议、工具调用展开、中断与追问、项目空间五大能力,是用户感知 Automy 价值的"门面"。

    模块概述

    工作台是用户使用 Automy 的主界面,投资公司员工常同时处理多个尽调项目、研究主题,需要在多个会话间快速切换、对比、引用。工作台必须支持流畅的多会话管理,并让 Agent 的"思考过程"——包括调用哪些工具、检索了哪些知识、使用了哪些 MCP 服务——对用户透明可见,既增加信任度,也便于调试与追溯。

    本模块围绕"会话"这一核心抽象展开,从创建、流式交互、工具调用展示、中断追问,到归档与项目归类,完整覆盖会话生命周期的所有交互需求。同时引入"项目空间"概念,把相关会话、文档、记忆组织为统一的工作上下文。

    需求条目

    REQ-3.1 多会话管理
    P0

    系统 SHALL 提供会话列表、创建、切换、归档、搜索、置顶等全生命周期管理能力;SHALL 支持按项目、按专家、按时间三种分组视图;同一用户 SHALL 可同时打开多个会话并通过侧边栏快速切换。

    • 创建会话:可选择专家 Agent 或空白会话开始,支持从模板创建(如 "尽调分析模板")
    • 会话列表:侧边栏展示历史会话,默认按最近活动时间倒序,支持搜索、置顶、归档
    • 会话分组:支持按项目、按专家、按时间三种分组视图,用户可切换
    • 会话命名:支持手动重命名,首次消息发出后 AI 自动生成建议命名(可采纳或修改)
    • 会话归档:已完成或暂时不用的会话可归档,归档后从主列表移除但可检索恢复
    • 会话搜索:支持按关键词、时间范围、专家、项目检索历史会话
    • 会话导出:单个会话或批量导出为 Markdown / PDF,保留消息结构

    场景

    WHEN 投资经理张三同时处理 3 个项目:ABC 新能源尽调、PQR 消费行业研究、季度策略报告
    THEN 张三在工作台创建 3 个会话并归入对应项目空间,切换到 "ABC 新能源尽调" 项目时,侧边栏自动过滤出该项目下的所有会话;切换会话时上下文立即恢复,无需等待加载。

    REQ-3.2 流式输出
    P0

    系统 SHALL 基于 SSE(Server-Sent Events)或 WebSocket 协议实现流式输出,LLM 边生成边显示,首 Token 响应时延 SHALL 不超过 1.5 秒;SHALL 支持打字机效果、消息气泡分段渲染、Markdown 实时解析(代码块、表格、公式)。

    • 协议选择:优先 SSE(单向流式,兼容性好),复杂交互场景使用 WebSocket
    • 首 Token 时延:P95 ≤ 1.5 秒,超时显示 "正在思考..." 占位
    • 打字机效果:平滑的字符级输出动画,可关闭以追求极致性能
    • 分段渲染:支持思考过程、正文回答、引用来源、工具调用分段展示
    • Markdown 实时解析:代码块语法高亮、表格、公式、图片实时渲染
    • 断线重连:网络中断后自动重连,从断点续传未完成的内容
    • 多模态:支持流式输出图片、表格、图表(基于 ECharts / Mermaid)

    场景

    WHEN 张三提问 "分析 ABC 新能源 2025 年半年报"
    THEN 专家在 1.2 秒内开始流式输出,首先展示 "正在检索 ABC 新能源财报...",随后分段输出:思考过程(灰色斜体)→ 正文回答(正常字体)→ 数据表格(实时渲染)→ 引用来源(脚注);整个过程张三可随时中断。

    REQ-3.3 工具调用展开
    P0

    系统 SHALL 实时展示 Agent 调用工具的过程,包括工具名称、输入参数、执行状态、返回结果(可折叠),让用户理解 Agent 的"思考路径";SHALL 支持工具调用失败的错误展示与重试入口。

    • 调用展示:工具调用以卡片形式嵌入消息流,展示工具名、状态(进行中/成功/失败)、耗时
    • 参数与结果:默认折叠,点击展开查看输入参数与返回结果(JSON 格式化展示)
    • 状态图标:进行中(转圈)/ 成功(绿勾)/ 失败(红叉)/ 已拦截(黄盾)
    • 失败重试:工具调用失败时展示错误原因,提供 "重试" 按钮与 "查看详情" 入口
    • 敏感操作:涉及写操作(如发送邮件、修改 CRM)的工具调用 SHALL 展示 "确认" 按钮,用户确认后才执行
    • 调用链路:多步骤工具调用展示为调用链,可视化 Agent 的决策路径

    场景

    WHEN 张三问 "查一下本月 ABC 新能源相关的所有舆情,并总结风险点"
    THEN 专家依次调用 knowledge.search(检索内部研究库)→ mcp.wind.query(查询 Wind 数据)→ mcp.email.search(搜索邮件);每个工具调用以卡片展示,张三可点击展开查看具体参数与返回结果,最终在正文给出风险总结。
    WHEN 专家准备调用 mcp.crm.update 修改客户信息
    THEN 工具调用卡片展示 "需要确认" 状态,展示将修改的字段与值,张三点击 "确认" 后才执行,执行结果实时反馈。

    REQ-3.4 中断与追问
    P1

    系统 SHALL 支持用户随时中断当前生成、追问澄清、回滚到指定消息重新生成;中断 SHALL 保留已生成内容,追问 SHALL 在当前上下文继续,回滚 SHALL 创建新的分支以保留原始对话轨迹。

    • 中断:点击 "停止" 按钮或按 Esc 中断当前生成,已生成内容保留并标记为"已中断"
    • 追问:在已生成消息下方直接输入追问,无需新开会话,上下文延续
    • 回滚重试:任意消息可 "回滚到此处重新生成",系统创建分支保留原版本,用户可对比
    • 分支管理:同一会话可有多条分支,用户可切换查看不同版本的回答
    • 编辑重发:支持编辑已发送消息后重发,系统创建新分支
    • 引用追问:可引用某条历史消息进行追问,生成 "基于你之前说的 X,那么 Y?" 的提问

    场景

    WHEN 张三发现专家正在生成的回答偏离主题
    THEN 张三点击 "停止" 中断生成,已生成的部分内容保留并标记灰色 "已中断";张三在追问框输入 "请聚焦在动力电池业务,不要展开整车业务",专家在新消息中调整回答方向。
    WHEN 张三想尝试用不同模型重新生成同一条回答
    THEN 张三点击消息的 "回滚重试",选择 GLM-5 模型,系统创建分支保留原 DeepSeek 回答,新分支展示 GLM-5 版本,张三可并排对比两个版本的回答。

    REQ-3.5 项目空间
    P1

    系统 SHALL 支持按项目组织会话与文档,每个项目独立维护会话列表、文档库、项目记忆、项目成员,确保跨会话的项目上下文延续与团队协作;SHALL 提供项目看板展示进度、待办、关键决策。

    • 项目隔离:每个项目独立的会话列表、文档库、记忆空间、成员列表
    • 项目成员:支持邀请多人加入项目,成员可共享会话、文档、项目记忆
    • 项目文件:项目相关文档统一管理,支持上传、关联知识库文档
    • 项目看板:展示项目进度、待办事项、关键决策记录、最近活动
    • 跨会话延续:新会话归属项目后,自动加载项目记忆与文档作为上下文
    • 项目归档:项目完成后可归档,归档后项目数据保留但不再活跃显示
    • 权限继承:项目成员继承项目内会话与文档的访问权限

    场景

    WHEN 张三创建 "ABC 新能源尽调" 项目空间,邀请同事李四、王五加入
    THEN 三人共享该项目下的所有会话与文档,张三与专家的对话结论自动写入项目记忆,李四新开会话时自动加载 ABC 项目背景;项目看板展示 "尽调进度:资料收集 → 财务分析 → 风险评估 → 投决建议"。

    富媒体输入支持

    📎 文件上传

    PDF、Word、Excel、PPT、图片,单文件 ≤ 100MB,自动提取文本与表格

    🎙 语音输入

    语音转文字后发送,支持中英文混合,自动标点

    💻 代码块

    支持代码粘贴与高亮,Python / SQL / JavaScript 等 20+ 语言

    @ 提及

    @专家 / @技能 / @文档 / @记忆 快速引用上下文

    验收标准

    • ✅ 多会话流畅切换,首屏加载 ≤ 2 秒
    • ✅ 流式输出首 Token ≤ 1.5 秒,断线自动重连
    • ✅ 工具调用过程透明展示,可折叠查看参数与结果
    • ✅ 中断、追问、回滚重试均正常工作,保留分支历史
    • ✅ 项目空间支持多人协作,记忆与文档跨会话延续
