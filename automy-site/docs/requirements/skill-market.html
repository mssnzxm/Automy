    • 技能市场

    提供可复用、可组合的技能生态。用户可低代码自定义技能、从公共市场一键安装、团队内共享,大幅降低 Agent 能力构建成本,让最佳实践(财报解析、合同审查、商机查询)在组织内沉淀与传播。

    ⚡
    模块定位
    本模块定义技能定义模型、低代码编辑器、运行时沙箱、市场浏览、安装与依赖管理、审核与下架六大核心能力,覆盖技能从定义、开发、测试、发布、安装、运行到下架的完整生命周期。

    模块概述

    投资公司内部存在大量可复用的工作流程:财报解析、合同审查、商机查询、会议纪要、监管动态跟踪等。如果没有技能抽象,每个用户都要从零配置 Agent,效率低下且能力无法沉淀,最佳实践难以在团队内传播。

    Automy 技能市场让最佳实践可被复用与传播:投研骨干把 "财报解析" 流程封装为技能发布到团队市场,新人一键安装即可获得同等能力;法务专家把 "合同风险审查" 技能发布到公司市场,全员可用。技能支持低代码可视化编排、在线调试、版本管理、运行时沙箱隔离,确保安全可控。

    需求条目

    REQ-4.1 技能定义模型
    P0

    系统 SHALL 提供统一的技能定义模型,每个技能 SHALL 包含元数据、提示词、参数 Schema、工具集、工作流、权限声明六大组成部分;定义 SHALL 同时支持 YAML 声明式与可视化 JSON 两种格式,可互转。

    组成部分说明示例
    元数据name / version / category / description / author / iconname: 财务报表解析, version: 2.1.0
    提示词system prompt 与 user prompt 模板,支持变量插值"你是资深财务分析师,请分析 {{company}} 的 {{period}} 财报..."
    参数 SchemaJSON Schema 定义输入参数document(file) / period(string) / metrics(array)
    工具集技能可调用的工具清单pdf.extract / knowledge.search / formula.compute
    工作流多步骤编排,支持条件分支与循环step1 提取 → step2 识别 → step3 对比
    权限声明技能所需的 MCP 服务、知识库、记忆范围需要 wind-data / 内部研究库 / 写入项目记忆

    技能结构示例

    YAML 
    name: 财务报表解析
version: 2.1.0
category: 投研
description: 自动解析企业年度财务报表

input:
  document:
    type: file
    accept: [.pdf, .xlsx]

tools:
  - name: pdf.extract
  - name: knowledge.search
  - name: formula.compute

workflow:
  - step: 提取文档文本
  - step: 识别财务指标
  - step: 对比历史数据
  - step: 输出结构化摘要

    场景

    WHEN 投研骨干创建 "财务报表解析" 技能
    THEN 系统校验技能定义模型完整性:必须包含 name、version、input、workflow;tools 与 permissions 可选但建议填写;校验通过后保存为草稿,可进入编辑器继续完善。

    REQ-4.2 技能编辑器
    P0

    系统 SHALL 提供低代码可视化技能编辑器,支持拖拽式工作流编排、模板编写、在线调试、版本管理;SHALL 同时支持 YAML / JSON 源码编辑模式,可视化与源码双向同步。

    • 拖拽编排:可视化画布拖拽节点(工具、条件、循环、LLM 调用、知识检索),连线定义数据流
    • 模板编写:内置技能模板库(财报分析、合同审查、会议纪要等),基于模板快速创建
    • 在线调试:提供测试环境,可输入测试数据逐步执行技能,查看每步输入输出
    • 断点调试:在工作流节点设置断点,单步执行,检查中间变量
    • 版本管理:每次保存创建新版本,支持版本对比、回滚、灰度发布
    • 源码模式:支持 YAML / JSON 源码编辑,与可视化画布双向同步
    • 调试日志:记录每次调试的输入、输出、耗时、Token 消耗

    场景

    WHEN 投研骨干张三在编辑器中调试 "财务报表解析" 技能
    THEN 张三上传一份测试财报 PDF,点击 "调试" 按钮,系统逐步执行工作流:step1 提取文本(展示提取结果)→ step2 识别财务指标(高亮识别的指标)→ step3 对比历史数据(展示对比表格)→ step4 输出摘要;每步耗时与 Token 消耗实时显示。

    REQ-4.3 技能运行时沙箱
    P0

    系统 SHALL 在隔离的沙箱环境中执行技能,限制文件访问、网络请求、内存与 CPU 占用、执行时长;SHALL 禁止技能直接访问宿主文件系统与其他技能数据,确保恶意或异常技能不影响平台稳定性。

    • 文件隔离:技能仅能访问授权的临时工作目录,禁止访问宿主文件系统
    • 网络限制:仅允许调用声明中授权的 MCP 服务与外部 API,默认禁止其他网络请求
    • 资源限制:单次执行内存 ≤ 512MB,CPU ≤ 1 核,执行时长 ≤ 60 秒(可配置)
    • 权限校验:运行前校验技能声明的权限是否被用户授权,未授权的调用被拦截
    • 异常处理:技能抛出未捕获异常时,沙箱捕获并优雅降级,返回友好错误信息
    • 审计记录:沙箱内所有工具调用、文件访问、网络请求均记录到审计日志

    场景

    WHEN 一个第三方技能尝试访问 /etc/passwd 文件
    THEN 沙箱拦截该文件访问,返回 "权限拒绝:技能无权访问该路径",并记录安全审计事件 "技能尝试越权文件访问",管理员可在审计日志中查看并下架该技能。
    WHEN 技能 "财务报表解析" 执行超时(超过 60 秒)
    THEN 沙箱强制终止技能执行,返回 "执行超时,请优化技能或减小输入数据量",已执行的部分结果保留,用户可查看执行日志定位瓶颈步骤。

    REQ-4.4 技能市场浏览
    P0

    系统 SHALL 提供技能市场浏览界面,支持按分类浏览、关键词搜索、评分排序、使用量排序;每个技能 SHALL 展示详情页,包含功能说明、参数说明、所需权限、版本历史、用户评价、使用统计。

    • 分类浏览:按投研 / 法务 / 合规 / 销售 / 通用分类,支持多级子分类
    • 关键词搜索:全文搜索技能名称、描述、标签,支持拼音与同义词
    • 排序:按评分、使用量、最新发布、最近更新多种排序方式
    • 技能详情页:功能说明、参数说明、所需权限、示例输入输出、版本历史、用户评价
    • 提供方标识:区分官方 / 团队 / 社区技能,官方技能标记 "已认证" 徽章
    • 使用统计:展示安装数、调用量、平均评分,辅助用户决策

    场景

    WHEN 新员工王五想找财报分析的技能
    THEN 王五在市场搜索 "财报",系统返回 5 个相关技能,按使用量排序展示;"财务报表解析 v2.1" 排在首位,展示安装数 234、评分 4.8、提供方 "官方 + 投研团队";王五点击进入详情页查看参数说明与示例输出,确认符合需求后点击 "安装"。

    REQ-4.5 技能安装与依赖
    P0

    系统 SHALL 支持一键安装技能到个人空间,安装时 SHALL 解析技能依赖的 MCP 服务、知识库、其他技能,并明确提示用户授权;SHALL 支持技能更新通知与一键升级。

    • 一键安装:点击 "安装" 按钮即装即用,无需复杂配置
    • 依赖解析:自动解析技能声明的依赖(MCP 服务、知识库、其他技能)
    • 授权提示:安装时弹窗展示技能所需的权限清单,用户逐项授权后才完成安装
    • 依赖检查:检查依赖的 MCP 服务用户是否已连接,未连接的提示先连接
    • 更新通知:技能发布新版本时,已安装用户收到更新通知
    • 一键升级:支持一键升级到最新版本,保留个人配置
    • 卸载清理:卸载技能时清理相关配置,但保留已产生的会话数据

    场景

    WHEN 王五安装 "财务报表解析" 技能
    THEN 系统解析依赖:(1) MCP 服务 wind-data;(2) 知识库 内部研究库;(3) 技能 pdf-text-extract;弹窗展示权限清单 "需要访问 Wind 数据、内部研究库、PDF 提取工具",王五逐项授权后安装完成。
    WHEN 王五未连接 wind-data MCP 服务
    THEN 安装弹窗提示 "该技能依赖 Wind 数据 MCP 服务,您尚未连接,请先连接",并提供跳转到 MCP 服务目录的链接;王五可先完成连接再回来安装。

    REQ-4.6 技能审核与下架
    P1

    系统 SHALL 对发布到团队市场与公司市场的技能进行管理员审核,审核内容包括功能合规性、安全扫描、权限合理性;SHALL 支持紧急下架违规技能,已安装用户收到通知并自动停用。

    • 审核流程:作者提交发布申请 → 管理员审核(功能 + 安全 + 权限)→ 通过 / 驳回 / 待修改
    • 安全扫描:自动扫描技能代码与提示词,检测恶意代码、提示词注入、越权访问
    • 权限审查:审查技能声明的权限是否最小化,过度授权的技能驳回
    • 紧急下架:管理员可一键下架违规技能,已安装用户立即收到通知,技能自动停用
    • 申诉机制:作者对驳回决定可申诉,二次审核由更高级别管理员处理
    • 审核日志:所有审核操作记录到审计日志,可追溯

    场景

    WHEN 管理员发现 "客户洞察分析" 技能存在提示词注入风险
    THEN 管理员点击 "紧急下架",系统立即停用该技能,234 名已安装用户收到站内信通知 "您安装的 '客户洞察分析' 技能因安全原因已被下架,请联系管理员",技能在市场不可见。
