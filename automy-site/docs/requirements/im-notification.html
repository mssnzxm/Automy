    • IM 通道与主动提醒

    通过企业微信、飞书、钉钉三大 IM 通道接入,让 Automy 从被动应答升级为主动触达与双向交互,在员工熟悉的工作场景中完成提醒、回调与跨端协同,让 Agent 真正融入投研、风控、合规的日常工作流。

    📨
    模块定位
    本模块定义 IM 通道接入、统一通知服务、主动提醒引擎、通知模板、双向交互回调与通知权限隐私六项核心能力,支撑投资公司会议提醒、审批待办、任务完成触达、群内 @Agent 查 CRM 等高频场景。

    模块概述

    投资公司员工日常高频使用企业微信、飞书、钉钉进行沟通协作,而传统智能体仅能在独立 Web 工作台使用,存在"打开成本高、被动等用户、消息孤岛"等问题。Automy 通过IM 通道接入 + 统一通知服务 + 主动提醒引擎三层架构打通这一鸿沟:上游业务事件(任务完成、文档更新、会议开始、审批待办、记忆更新)通过统一 API 进入通知服务,经用户偏好过滤与节流后路由到飞书 / 企微 / 钉钉 / 站内信 / 邮件,IM 内 @应用或点击卡片按钮可回调到 Agent 完成双向交互,会话上下文与 Web 工作台打通实现跨端续接。

    需求条目

    REQ-6.1 IM 通道接入
    P0

    系统 SHALL 支持企业微信(WeCom)、飞书(Lark)、钉钉(DingTalk)三大 IM 通道接入,每个通道 SHALL 完整配置 AppKey / AppSecret / 回调地址 / 可见范围 / 频率限制五项参数,并 SHALL 支持多租户隔离与通道级开关。

    • 通道配置:AppKey / AppSecret(KMS 加密存储)/ 回调 URL(签名校验)/ 可见范围(部门 / 项目)/ 频率限制(默认每用户每分钟 30 条)
    • 多租户隔离:同一通道下不同部门可见性隔离,投研部与合规部互不可见
    • 通道开关:管理员可独立启停某通道,不影响其他通道运行
    • 降级备份:通道不可用时自动降级到站内信,并写入告警日志
    • 凭证轮换:支持 AppSecret 90 天轮换提醒,不影响在线服务

    场景

    WHEN 投资公司 IT 管理员张工在管理后台开启飞书通道,填写 AppID / AppSecret / 事件订阅 URL,并将可见范围设为投研部与风控部
    THEN 系统调用飞书开放平台校验凭证有效性,回调 URL 通过签名校验后通道状态置为"已连接",投研部与风控部员工可收到 Automy 主动提醒,合规部员工不可见该通道。

    REQ-6.2 统一通知服务
    P0

    系统 SHALL 提供统一通知服务(Notification Service)屏蔽各 IM 平台差异,上游业务系统通过统一 API 调用,下游由路由层决定投递到站内信 / 企微 / 飞书 / 钉钉 / 邮件;SHALL 支持去重、合并与降级三种治理能力,避免重复打扰。

    • 上游统一 API:notify(userId, scene, payload, channels?, priority?)
    • 路由策略:用户偏好 → 渠道选择 → 降级链(飞书 → 企微 → 站内信)
    • 去重:同场景同内容 5 分钟内仅投递一次,合并为聚合卡片
    • 合并:同用户多场景通知 1 分钟窗口内合并为一条卡片
    • 降级:目标渠道失败率超阈值(默认 10%)自动降级到下一渠道
    • 优先级:P0 立即投递、P1 入队延迟投递、P2 仅站内信

    场景

    WHEN 投资经理李四的 ABC 新能源尽调报告生成任务完成,触发 notify(user_id, scene="report.completed", payload={reportId, title})
    THEN 通知服务读取李四偏好:首选飞书 + 备选站内信,渲染任务完成卡片模板,通过飞书 Adapter 投递卡片消息;若飞书连续失败 3 次则降级到站内信,并在审计日志记录降级原因。

    REQ-6.3 主动提醒引擎
    P0

    系统 SHALL 提供主动提醒引擎,支持场景触发与时间触发两类触发源,SHALL 通过规则编排将触发事件映射到模板与渠道,SHALL 实现节流策略(每用户每日上限、静默时段、聚合)避免打扰。

    • 场景触发:任务完成 / 文档更新 / 会议开始 / 审批待办 / 记忆更新五类内置场景
    • 时间触发:日程提醒(按日历事件)、日报提醒(每日定时)、回访提醒(客户 N 天未联系)
    • 规则编排:scene → template_id → channels → throttle_policy 四段式可配置
    • 节流策略:每用户每日上限默认 20 条、静默时段(默认 21:00-08:00 仅紧急)、相同场景 1 小时内合并
    • 规则热更新:管理员可在后台编辑规则,无需重启服务
    • 触发审计:每条触发记录来源、命中规则、最终投递结果

    场景

    WHEN 投资经理张三的日历显示 10:00 有 ABC 新能源投决会,系统在 09:45 触发时间提醒
    THEN 提醒引擎匹配规则"会议开始前 15 分钟",渲染模板"会议即将开始:ABC 新能源投决会 @ 10:00 会议室 A301",按张三偏好投递到飞书;若张三已设置该时段静默则仅站内信,并在工作台顶部展示待办。

    REQ-6.4 通知模板与变量插值
    P1

    系统 SHALL 提供通知模板管理能力,每条模板 SHALL 包含触发场景、渠道适配(飞书卡片 / 企微文本 / 钉钉 Markdown)、标题、正文、按钮、变量五要素,SHALL 支持多语言与在线试发预览。

    • 模板字段:scene / channel_layout / title / body / buttons / variables
    • 变量插值:{{user_name}} / {{project_name}} / {{report_title}} / {{meeting_time}} 等
    • 渠道适配:同一模板为飞书卡片 / 企微文本 / 钉钉 Markdown / 邮件 HTML 提供不同 layout
    • 多语言:支持中英文双语,按用户偏好自动切换
    • 在线试发:编辑器内选择场景与示例数据,实时预览渲染效果并支持测试投递
    • 版本管理:模板修改走版本审核,灰度发布

    场景

    WHEN 产品经理在通知模板编辑器新建"投决会即将开始"模板,变量 {{user_name}} / {{meeting_title}} / {{meeting_time}} / {{room}},点击"试发"
    THEN 系统使用示例数据渲染,展示飞书卡片、企微文本、钉钉 Markdown 三种渠道的预览效果,确认无误后保存为 v1 版本,经管理员审核后灰度发布到投研部。

    REQ-6.5 双向交互
    P1

    系统 SHALL 支持 IM 内 @应用与点击卡片按钮回调两种交互方式,SHALL 将回调请求路由到 Agent 编排引擎处理,SHALL 将处理结果回传到 IM;会话上下文 SHALL 与 Web 工作台打通,实现跨端续接。

    • @应用交互:IM 群内 @Automy 后输入指令,触发 Webhook 回调,进入 Agent 编排
    • 卡片按钮回调:通知卡片按钮点击触发 action callback,可携带上下文参数
    • 会话路由:回调消息关联到已有 Session 或创建新 Session,标注 source_channel
    • 跨端续接:IM 发起的会话可在 Web 工作台继续,显示来源标注"来自飞书"
    • 结果回传:Agent 输出渲染为 IM 卡片,支持多轮交互
    • 权限校验:回调携带用户身份,通过 SSO Token 校验后放行

    场景

    WHEN 投资经理李四在飞书群 @Automy 查询"ABC 新能源最新估值"
    THEN Webhook 接收回调,会话路由找到李四的活跃 Session(或新建),Agent 调用 CRM MCP 工具拉取数据,渲染为飞书卡片返回群内;同步在 Web 工作台显示该会话,标注"来源:飞书",李四可在 Web 继续追问。

    REQ-6.6 通知权限与隐私
    P1

    系统 SHALL 提供用户通知偏好设置(渠道开关、静默时段、聚合策略),SHALL 对涉密内容强制仅站内信投递,SHALL 对 IM 调用全链路审计,记录用户、场景、渠道、内容摘要与投递结果。

    • 用户偏好:渠道开关(飞书 / 企微 / 钉钉 / 邮件独立启停)、静默时段、聚合粒度
    • 涉密降级:涉密文档标记 L3 / L4 时,通知仅站内信,不投递 IM
    • 内容过滤:涉密关键词(项目代号、未公开财务数据)自动脱敏后再投递 IM
    • 全链路审计:记录 operator / scene / channel / template / status / sent_at / interacted_at
    • 用户视角:用户可在用户中心查看自己的通知历史与已读状态
    • 合规视角:合规审计员可按用户 / 时间 / 场景查询审计日志,导出报告

    场景

    WHEN ABC 新能源尽调报告涉密等级为 L3,系统需通知投资经理张三报告已完成
    THEN 通知服务识别 payload 涉密等级 L3,跳过飞书与企微渠道,仅在站内信投递"ABC 新能源尽调报告已完成,请前往工作台查看",并在审计日志记录"涉密降级到站内信"原因。
    WHEN 张三在用户中心关闭"钉钉通道",并设置静默时段 20:00-08:00
    THEN 系统立即生效,后续钉钉通道通知不再投递,非紧急通知在静默时段内累积到次日 08:00 后批量投递,紧急通知(P0)仍投递但标注"静默时段紧急"。

    关键交互流程

    主动提醒触发到投递流程
    1. 触发源(任务完成 / 日历事件 / 审批)产生事件
    2. 主动提醒引擎接收事件,匹配规则(scene → template → channels → throttle)
    3. 节流检查:超过每用户每日上限或处于静默时段则入队或丢弃
    4. 模板渲染:加载模板并完成变量插值与渠道适配
    5. 通知服务路由:用户偏好 → 渠道选择 → 降级链
    6. 通道适配器投递:飞书 / 企微 / 钉钉 Adapter 调用 IM 平台 API
    7. 用户在 IM 收到通知,点击卡片按钮触发回调
    8. 回调路由到 Agent,处理结果回传 IM 与 Web 工作台
    双向交互回调流程
    1. 用户在 IM 群 @Automy 或点击通知卡片按钮
    2. IM 平台将事件回调到 Automy Webhook 接收器
    3. Webhook 校验签名与用户身份
    4. 会话路由:找到已有 Session 或创建新 Session,标注 source_channel
    5. Agent 编排引擎(AgentScope Java 2.0)处理请求,调用 MCP 工具
    6. 结果渲染为 IM 卡片返回,同步到 Web 工作台
    7. 用户可在 IM 或 Web 继续多轮对话,上下文打通

    验收标准

    • ✅ 三大 IM 通道(企微 / 飞书 / 钉钉)可独立接入与启停
    • ✅ 统一通知 API 屏蔽 IM 差异,上游无感知
    • ✅ 主动提醒支持场景与时间两类触发,节流生效
    • ✅ 模板支持变量插值、多语言、在线试发
    • ✅ IM 内 @应用与卡片按钮可回调到 Agent
    • ✅ 会话上下文与 Web 工作台打通,跨端续接
    • ✅ 涉密内容自动降级到站内信
    • ✅ IM 调用全链路审计可追溯
