    核心概念

    Agent 智能体

    能够自主感知环境、做出决策并执行动作的软件实体。在 Automy 中,Agent 由 LLM 驱动,可调用工具、技能与记忆,完成复杂任务。

    AI智能体

    Skill 技能

    封装特定能力的可复用模块,可被 Agent 调用。技能可调用工具、访问知识库、串联其他技能,支持可视化编排。

    能力复用

    Expert Agent 专家智能体

    在特定领域(投研、法务、合规等)具备深度专业能力的预置 Agent,用户可订阅即用。

    专家订阅

    Long-term Memory 长期记忆

    跨会话持久化的用户偏好、项目上下文、决策记录等。让 Agent 真正"记住"用户,提供个性化服务。

    记忆持久化

    Session 会话

    用户与 Agent 的一次交互过程,包含上下文与历史消息。会话内的记忆默认不跨会话,需主动持久化。

    会话上下文

    Knowledge Base 知识库

    存储企业内部文档、研究资料、历史决策等结构化与非结构化知识,通过 RAG 提供检索增强。

    知识检索

    技术框架

    AgentScope Java 2.0

    阿里达摩院开源的 Java(JDK 17+)企业级 Agent 框架,基于 Project Reactor 响应式,提供 ReActAgent 推理内核 + Harness 工程化层 + 多 Agent 编排,原生支持 MCP/A2A 协议。Automy 采用 AgentScope Java 2.0 作为智能体编排引擎。

    框架Java阿里达摩院

    Eino 已废弃

    字节跳动开源的 LLM 应用开发框架。已废弃,Automy 已迁移至 AgentScope Java 2.0,详见智能体运行时章节。

    ReActAgent

    AgentScope Java 2.0 的推理内核,基于 ReAct(Reasoning + Acting)循环 + Project Reactor 响应式流,实现自主规划与工具调用。

    推理内核ReAct

    Harness 工程化层

    AgentScope Java 2.0 的企业级工程化底座,包含 Workspace / Session / Memory / Sandbox / Skill / Subagent 六大组件,通过 Hook 注入推理循环,不修改核心逻辑。

    工程化企业级

    Supervisor 多 Agent

    AgentScope Java 2.0 的多 Agent 编排模式,Leader-Worker 架构,通过 Markdown spec 声明 Subagent,支持同步/异步委派,替代传统 DAG 图编排。

    多 Agent编排

    Workspace 工作空间

    AgentScope Java 2.0 的结构化工作空间,作为 single source of truth,含 AGENTS.md / MEMORY.md / knowledge/ / skills/ / subagents/ 目录。

    工作空间SSOT

    @Tool 注解

    AgentScope Java 的工具定义方式,标记方法后框架自动生成 JSON Schema 注入提示,配合 Toolkit 统一管理本地工具与远端 MCP 工具。

    工具注解

    RAG Retrieval-Augmented Generation

    检索增强生成,先从知识库检索相关文档,再将其作为上下文输入 LLM 生成回答,提升准确性与可溯源性。

    Embedding 嵌入

    将文本转换为高维向量表示,用于计算语义相似度与检索。是 RAG 系统的核心组件。

    Streaming Output 流式输出

    LLM 边生成边返回结果,而非等待完整结果再返回,提升用户体验与感知速度。

    Tool Calling 工具调用

    LLM 通过结构化方式调用外部工具(函数、API),实现与外部世界交互。是 Agent 的核心能力之一。

    Workflow 工作流

    将复杂任务分解为多个有序步骤,每步可调用不同 Agent、技能或工具,支持条件分支与循环。

    协议标准

    MCP Model Context Protocol

    模型上下文协议,由 Anthropic 主导的开放标准,定义 LLM 应用如何与外部数据源、工具、服务交互。Automy 通过 MCP 接入企业系统。

    MCP 基于 JSON-RPC 2.0,核心概念:Server(服务端)、Client(客户端)、Resource(资源)、Tool(工具)、Prompt(提示模板)。

    A2A 协议 Agent-to-Agent Protocol

    Agent-to-Agent 协议,Agent 可注册到 Nacos,其他 Agent 可像调用微服务一样发现并委派任务,跨进程跨机调用。AgentScope Java 2.0 原生支持 A2A。

    协议AgentNacos

    安全合规

    RBAC Role-Based Access Control

    基于角色的访问控制,通过角色分配权限,简化权限管理。Automy 支持用户、角色、权限三级管理。

    Audit Log 审计日志

    记录用户与 Agent 的所有操作行为,包括调用、数据访问、配置变更等,用于合规追溯。

    Private Deployment 私有化部署

    将软件完整部署到企业内网,数据不出企业边界,满足数据主权与合规要求。

    Data Sovereignty 数据主权

    数据完全归企业所有与控制,包括存储、处理、传输的全链路。是 Automy 的核心设计原则。
